# Deliverability Audit Checklist

## Phase 1: Setup

### Create Fresh Test Accounts

Create brand-new email accounts on each major provider. These accounts must have **never received email from your brand** -- no whitelisting, no "not spam" clicks.

- Gmail (Google)
- Outlook/Hotmail (Microsoft)
- Yahoo/AOL
- Zoho Mail
- Mail.com
- iCloud (Apple)

### Gather Access

- DNS hosting (Cloudflare, GoDaddy, Namecheap)
- Email marketing platform (ActiveCampaign, Klaviyo, Mailchimp)
- Cold email sending tool (Lemlist, Instantly, Mailshake)
- Google Workspace admin console

### Inventory All Email Senders

Compile a list of EVERY platform sending email as your domain:
- Marketing ESP
- Cold email tool
- Corporate email (Google Workspace, Zoho)
- Transactional email (SendGrid, SocketLabs, Mailgun)
- Website/ecommerce platform
- CRM system
- Any other notification system

---

## Phase 2: Provider Tests

### Send Test Emails

Send at least 10 sample emails across the test mailboxes. Include a mix of:
- Cold outreach emails (your actual templates)
- Any automated emails (welcome, follow-up sequences)
- Different "from" addresses if you use multiple

### Log Results for Each Email

For every test email, record:
- Type (cold outreach / automation / transactional)
- Subject line
- From address
- Style (plain text / HTML)
- Inbox placement per provider: **Inbox / Promotions / Spam / Not Received**

**Color coding:**
- Green = Inbox (primary)
- Yellow = Promotions tab
- Red = Spam folder
- Black = Not received at all

## Phase 3: Authentication Audit

### Header Analysis (via Gmail)

1. Open test email in Gmail web interface
2. Click vertical ellipsis (three dots) > "Show Original"
3. Check three values:
   - **SPF:** PASS or FAIL
   - **DKIM:** PASS or FAIL
   - **DMARC:** PASS or FAIL
4. Check "Authenticated as":
   - PASS with domain `yourdomain.com` = **Authenticated as Self** (ideal)
   - PASS with domain `[provider-domain.com]` = **Authenticated as Provider** (fix needed)

### Blacklist Check

1. Go to MXToolbox.com/blacklists
2. Test your email domain
3. Test the IP address of your sending platform
4. Check for Spamcop specifically -- the one widely-regarded blacklist to worry about
5. Shared IP blacklisting is common on lower-tier ESP plans -- not always a crisis
6. Gmail and Yahoo use their own reputation management and don't rely heavily on third-party blacklists

## Phase 4: Content Testing

### Mail-Tester.com (Free)

1. Go to Mail-Tester.com
2. Copy the unique test email address
3. Send your cold email template to that address
4. Check the score

**Score interpretation:**
- 9+/10 = Good shape (most deductions from tracking pixel alt tags)
- 7-9/10 = Fixable issues (check alt tags, spam words, links)
- Below 7/10 = Serious issues requiring immediate attention

**Note:** ActiveCampaign's open-tracking pixel never has an alt tag -- ignore this specific flag.

### SendForensics ($49/month)

- Deeper content analysis and vocabulary scoring
- Identifies problematic phrases
- **Target: 85%+ = good, 75%+ = acceptable**
- Use during deliverability rehabilitation especially

### Content Audit Items

- [ ] All images have alt tags
- [ ] No JavaScript in email templates
- [ ] CSS is inline only (no external stylesheets, no `<style>` in `<head>`)
- [ ] Link density under 5% of total text (exclude unsubscribe link)
- [ ] No ALL CAPS words in subject lines
- [ ] Spam trigger words under 5% density
- [ ] Plain text fallback exists (auto-generated by most ESPs)
- [ ] Cold emails: under 150 words, plain text, no images, no links in first email

## Phase 5: Sending Strategy Audit

### For Cold Email

- [ ] Using separate domains (not primary)
- [ ] Volume per domain under 30/day
- [ ] Warmup running on every inbox
- [ ] Spintax implemented for variation
- [ ] Unsubscribe option included
- [ ] DNC list maintained and enforced
- [ ] Custom tracking domain configured
- [ ] Email signature is plain text (no images, no HTML)

### For Warm/Marketing Email (if applicable)

- [ ] Automation count (welcome, post-purchase, winback, sunset)
- [ ] Campaign frequency logged
- [ ] Content balance assessed (promotional vs. educational vs. conversational)
- [ ] Targeting balance assessed (segmented vs. full list blasts)
- [ ] Inactive subscribers excluded from campaign sends

---

## Phase 6: List Health Audit

### For Cold Email Lists

- [ ] Emails verified through NeverBounce or similar
- [ ] Bounce rate tracking in place
- [ ] Current bounce rate below 2%
- [ ] DNC list updated and cross-referenced before every send
- [ ] No purchased lists being sent from primary domain

### For Warm Email Lists (if applicable)

- [ ] Paid ad audience targeting refined (not broad)
- [ ] Ad copy accurately represents email experience
- [ ] reCAPTCHA on opt-in forms (v3 invisible, score threshold 0.5)
- [ ] Honeypot fields on opt-in forms
- [ ] Engagement segments created (engaged, disengaged, inactive)
- [ ] Inactive subscribers excluded from campaigns

---

## Phase 7: Monitoring Setup

### Google Postmaster Tools

- Register your domains and sending IPs
- Monitor: IP reputation, spam complaint rates, bounce rates, DMARC pass rates
- Requires SPF, DKIM, DMARC to be set up first
- Check weekly for reputation trends

### Yahoo Feedback Loop

- Register for spam complaint feedback on Yahoo/AOL delivery
- Raw data format -- harder to parse but valuable

### Microsoft SNDS (Smart Network Data Services)

- Monitor Outlook/Hotmail delivery reputation
- Check monthly

### MXToolbox Paid Monitoring (Optional)

- Aggregates all feedback loops into one dashboard
- Creates quality score across all providers
- Worth the investment at scale (50+ domains)

## Audit Output Template

After completing the audit, create a prioritized fix list:

**Priority 1 (Fix immediately):**
- Missing or failing SPF/DKIM/DMARC records
- Sending from primary domain
- No warmup running
- Bounce rate above 3%

**Priority 2 (Fix within 1 week):**
- Emails authenticating as provider instead of self
- Missing custom tracking domain
- No unsubscribe option
- Spam words in templates

**Priority 3 (Fix within 1 month):**
- Content optimization (alt tags, link density)
- Sending strategy improvements
- Monitoring tool setup
- List cleaning schedule

**Timeline for improvement after fixes:**
- Authentication fixes: Improvement visible within 1-2 weeks
- Warmup completion: 2-4 weeks
- Reputation recovery: 30-90 days depending on severity
