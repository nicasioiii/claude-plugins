# Rewrite Section

This command diagnoses issues in any page section and provides a rewritten version with clear improvements. Perfect for fixing weak headlines, unclear benefit statements, weak CTAs, or any underperforming copy.

## What You'll Get

- Diagnosis of what's wrong with the current section
- Specific issues identified (clarity, benefit, urgency, relevance, etc.)
- Rewritten section (2-3 variations)
- Explanation of improvements made
- A/B testing guidance

## Questions I'll Ask

1. **Paste or describe the section you want rewritten.**
   - Include existing copy, even if it's rough

2. **What type of section is this?**
   - Headline, subheading, opening, benefit statement, objection handler, CTA, testimonial, etc.

3. **What is the page selling?**
   - Product/service name and primary offer

4. **What should this section accomplish?**
   - Get attention, build credibility, answer objection, drive urgency, etc.

## My Process

Depending on the section type, I'll reference relevant skills:
- **headline-hook-engine** — If it's a headline or opening
- **copy-flow-organization** — If it's about structure or flow
- **call-to-action-mastery** — If it's a CTA or closing
- **objection-psychology-framework** — If it's addressing concerns
- **story-narrative-copywriting** — If it needs emotional hook
- **benefit-discovery** — If it's a benefit statement
- **big-idea-positioning** — If it's positioning copy

## Output Format

Your rewrite analysis will include:

### DIAGNOSIS
```
Current problems:
1. [Specific issue] — Why this hurts conversion
2. [Specific issue] — Why this hurts conversion
3. [Specific issue] — Why this hurts conversion
```

### REWRITTEN VERSIONS

**VERSION 1: [Approach/angle]**
[Improved copy]

**VERSION 2: [Different approach]**
[Improved copy]

### WHY THESE WORK BETTER
- Specific improvements made
- How they address the diagnosis
- Expected impact on conversions

### A/B TESTING NOTES
- Which version to test first
- What metrics to track
- How long to run test
