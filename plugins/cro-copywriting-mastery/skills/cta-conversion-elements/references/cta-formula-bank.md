# CTA Formula Bank

**Sources:** Copy Hackers (10x Landing Pages), Arman Assadi (7-Figure Copywriting), OnePeak ($1M LP Blueprint), Ryan Deiss (High-Converting Homepage)

---

## CTA Copy Formulas

### Formula 1: The "I Want" Button
Write button copy from the visitor's perspective using first person.

**Template:** "I want to [desirable action/outcome]"

| Example | Page Type |
|---------|-----------|
| I want to grow my list | SaaS landing page |
| I want better sleep tonight | Health product |
| I want my free strategy session | Service page |
| I want to double my conversions | Course/training |
| Show me my personalized plan | Assessment tool |
| Give me instant access | Digital product |
| Yes, I want this | General purpose |

**Rule:** Keep "me/my" in the copy. It feels awkward to write but consistently outperforms third-person alternatives in testing.

---

### Formula 2: The "Get" Button
"Get" as the first word signals low friction and good things coming.

**Template:** "Get [desirable thing]"

| Example | Context |
|---------|---------|
| Get my free guide | Lead magnet |
| Get instant access | Digital product |
| Get started for free | Freemium SaaS |
| Get your copy | Book/report |
| Get the toolkit | Resource bundle |
| Get pricing | B2B/enterprise |
| Get matched | Marketplace |

---

### Formula 3: Value-First CTA
Lead with the outcome, not the action.

**Template:** "[Achieve desirable outcome]"

| Value-First (stronger) | Action-First (weaker) |
|------------------------|----------------------|
| Generate more leads | Download ebook |
| Start saving 8 hours a week | Sign up |
| See your personalized plan | Submit |
| Boost my conversion rate | Create account |
| Grow my revenue | Join now |
| Fix my landing page | Get started |

**When to use:** Problem-aware and solution-aware visitors who need the VALUE articulated. They care more about what they GET than what they DO.

**Exception:** Most-aware, high-intent visitors respond better to direct action: "Buy now," "Add to cart," "Complete purchase." Don't get cute with hot traffic.

---

### Formula 4: Hook-Line-Sinker
Three components combined in one CTA.

**Template:** "[Command verb] + [Offer] + [Urgency]"

| Example | Breakdown |
|---------|-----------|
| Download the blueprint today | Command + Offer + Urgency |
| Start your free trial before midnight | Command + Offer + Urgency |
| Claim your spot -- only 12 left | Command + Offer + Scarcity |
| Reserve my seat now | Command + Offer + Urgency |
| Lock in this price today | Command + Offer + Urgency |

---

### Formula 5: RAD Check
Use this as a quality check before finalizing any CTA.

- **R -- Require:** Does the visitor have all the information needed to click?
- **A -- Acquire:** Is the button easy to find and click?
- **D -- Desire:** Does the visitor want what the button promises?

If any answer is "no," fix that element before launching.

---

### Formula 6: CPSC Button Audit
Final visual check for every button on the page.

- **C -- Copy:** Is the text clear about what happens after clicking?
- **P -- Placement:** Is the button where eyes naturally look?
- **S -- Size:** Is it large enough to be an easy click target?
- **C -- Color:** Does it contrast sharply with surrounding elements?

## CTA Strengthening Techniques

### Add Benefits to CTA
**Before:** "Sign up"
**After:** "Sign up and start growing your list today"

### Add Scarcity to CTA
**Before:** "Enroll now"
**After:** "Enroll now -- only 7 spots left this month"

### Add Social Proof to CTA
**Before:** "Get started"
**After:** "Join 47,000+ marketers -- get started free"

### Add Risk Reversal to CTA
**Before:** "Buy now"
**After:** "Buy now -- 60-day money-back guarantee"

## Reassuring Text Below Buttons

Place micro-copy directly below every CTA button to reduce anxiety.

### Removing Risk
- "No credit card required"
- "Cancel anytime -- no questions asked"
- "100% money-back guarantee"
- "Your information is secure and never shared"
- "Risk-free for 30 days"

### Setting Expectations
- "Next: tell us where to ship it"
- "You'll get instant access to your dashboard"
- "Next step: choose your plan"
- "Takes less than 60 seconds"
- "You'll receive a confirmation email immediately"

### Reducing Social Pressure
- "No salesperson will call"
- "No obligation to continue"
- "Unsubscribe at any time"
- "We respect your privacy"

### Adding Social Proof
- "Join 12,000+ happy customers"
- "Rated 4.9/5 by 2,300 users"
- "Trusted by teams at [company names]"

**Rule:** Reassuring text should NEVER suggest work is about to begin. "Next: fill out a quick form" is bad. "Next: see your personalized results" is good.

## DO/DON'T Button Word Lists

### Words That INCREASE Clicks

| Word | Why It Works |
|------|-------------|
| Get | Low friction, suggests receiving value |
| Start | Implies beginning of positive change |
| Claim | Implies ownership, something reserved for them |
| Join | Community, belonging |
| Discover | Curiosity, exploration |
| See | Low commitment, visual |
| Try | Low risk, experimental |
| Access | Exclusivity, unlocking |
| Unlock | Value behind a gate |
| Save | Direct benefit |
| Create | Empowerment, agency |
| Build | Progress, construction |
| Download | Tangible deliverable |
| Watch | Passive, easy consumption |
| Learn | Self-improvement |
| Show me | Curiosity + first person |
| Give me | Direct + first person |
| Yes | Affirmative, momentum |
| Free | Removes risk entirely |
| My/Me | Personal ownership |
| Now | Immediacy |
| Today | Urgency without pressure |
| Instantly | Speed of gratification |

### Words That DECREASE Clicks

| Word | Why It Fails |
|------|-------------|
| Submit | Implies subservience, forms, bureaucracy |
| Enter | Vague, institutional |
| Click here | Tells them what to do, not what they get |
| Continue | What am I continuing to? Vague |
| Proceed | Formal, bureaucratic |
| Register | Feels like paperwork |
| Sign up (alone) | No benefit attached |
| Buy (cold traffic) | Too direct for unaware/problem-aware visitors |
| Order | Sounds like work, restaurants |
| Pay | Reminds them of spending money |
| Apply | Implies they might be rejected |
| Process | Bureaucratic, slow |

## CTA by Page Type

### Landing Page (Opt-in)
- Primary: "Get my free [resource]" / "Send me the [resource]"
- Reassurance: "No spam. Unsubscribe anytime."

### Landing Page (Paid Product)
- Primary: "Get instant access" / "Start my [timeframe] trial"
- Reassurance: "30-day money-back guarantee. No risk."

### Sales Page
- First CTA (after lead): "Yes, I want [primary benefit]"
- Second CTA (after proof): "Get [product] now"
- Final CTA (after close): "Claim my copy before [deadline]"
- P.S. CTA: "Click here to [primary benefit] -- risk free"

### Homepage
- Primary: Matches awareness stage (see CTA hierarchy in SKILL.md)
- Secondary (ghost button): Lower-commitment alternative

### Email
- Primary link: Appears 2-3 times (top, middle, end of email)
- P.S. link: Always include a link in the P.S.
- Format: Text link (not image button) for deliverability

### Checkout Page
- Primary: "Complete my order" / "I'm ready to check out"
- Reassurance: "Secure checkout. Your data is encrypted."

## CTA Placement Rules

1. **Above the fold:** Only if visitor is product-aware or most-aware. Otherwise, omit.
2. **After social proof:** Always. Proof builds confidence; CTA captures it.
3. **After offer details:** Always. They need to know what they get before acting.
4. **After FAQ:** Always. FAQs resolve final objections; CTA captures the resolution.
5. **After P.S.:** Always (in sales pages and emails). P.S. is one of the most-read elements.

**Rule:** Every CTA should drive the SAME action. One page, one offer, one action. No backup offers. No competing CTAs.

## Testing CTAs

### What to Test (Priority Order)
1. Value proposition in button text (what they GET)
2. First person vs. third person ("Get my..." vs. "Get your...")
3. With vs. without reassuring text
4. Button color (for contrast, not "best color")
5. Button size (larger almost always wins)
6. CTA placement (add/remove specific locations)

### How to Test
- A/B test with minimum 1,000 visitors per variant
- Run for minimum 2 weeks (account for day-of-week patterns)
- Only test ONE element at a time
- Use the "So What?" test on every CTA variant: does it promise value?
