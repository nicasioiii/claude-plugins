# /homepage-structure

You are helping the user design a homepage structure. Follow this workflow:

## Step 1: Understand the Business

Ask the user:
1. **What type of business?** (SaaS, agency, e-commerce, portfolio, startup)
2. **What is the primary goal?** (sign-ups, purchases, bookings, leads, awareness)
3. **Who is the target audience?** (B2B, B2C, technical, general)
4. **What content is available?** (copy, images, testimonials, logos, case studies)
5. **What is the competitive landscape?** (luxury, budget, innovative, traditional)

## Step 2: Define Section Order

Recommend a section flow based on business type. Typical homepage anatomy:

| Order | Section | Purpose |
|-------|---------|---------|
| 1 | Navigation | Wayfinding, brand presence, primary CTA |
| 2 | Hero | First impression, value proposition, main CTA |
| 3 | Social proof bar | Trust (logos, stats, awards) |
| 4 | Features / Benefits | What the product does and why it matters |
| 5 | How it works | Process, steps, simplify complexity |
| 6 | Testimonials | Real user validation |
| 7 | Case studies / Portfolio | Proof of results |
| 8 | Pricing | Transparent value comparison |
| 9 | FAQ | Overcome objections |
| 10 | Final CTA | Last conversion opportunity |
| 11 | Footer | Navigation, legal, contact |

Not every homepage needs every section. Adapt to business type and content availability.

## Step 3: Design the Hero Section

The hero is the most critical section. Define:
- **Headline:** Clear value proposition (what + for whom + benefit)
- **Subheadline:** Supporting detail or differentiation
- **CTA:** Single primary action (high contrast button)
- **Visual:** Product screenshot, illustration, or hero image
- **Layout:** Left-aligned text + right visual, centered text + background, or full-bleed image

## Step 4: Plan Responsive Behavior

For each section, define:
- Desktop layout (12-column grid, max-width container)
- Tablet adjustments (6-column, stacked where needed)
- Mobile layout (single column, full-width CTAs, condensed nav)

Key responsive patterns:
- Navigation: horizontal → hamburger menu
- Multi-column grids: side-by-side → stacked
- Hero: text + image side-by-side → stacked with image below
- Pricing cards: row → horizontal scroll or stacked

## Step 5: Apply Pacing and Rhythm

- Alternate between dense information sections and breathing room
- Use consistent section padding from spacing system
- Create visual variety through alignment changes and background colors
- Build a cinematic scroll experience -- each section reveals the next part of the story
