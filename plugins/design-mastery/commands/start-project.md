# /start-project

You are launching a new design project. Follow this workflow:

## Step 1: Understand the Project

Ask the user these questions (adapt based on what they have already shared):

1. **What is the project?** (website, app, dashboard, Shopify store, logo, brand identity)
2. **Who is it for?** (target audience, client industry)
3. **What is the goal?** (business outcome, not just deliverable)
4. **What exists already?** (current site, brand guidelines, content)
5. **What is the timeline?** (hard deadlines, launch windows)
6. **What is the complexity?** (number of pages, novel IA, integrations, user roles)

## Step 2: Assess Complexity

Based on answers, classify using the complexity matrix from discovery-direction:
- **High:** Full discovery, IA, wireframes, then visual design
- **Medium:** Standard discovery, wireframes recommended
- **Low:** Light brief, jump to visual design

## Step 3: Recommend Process

Based on complexity, recommend the appropriate process depth and next steps. Always suggest the next skill in the workflow:

- If brand direction is unclear → recommend `/mood-board`
- If visual foundations needed → recommend `/type-system` or `/color-palette`
- If layout is the priority → recommend `/layout-grid`
- If it is a Shopify store → recommend `/shopify-store`
- If it is a logo project → recommend `/logo-process`

## Step 4: Provide Brief Template

Offer the 9-element brief extraction template from the skill so the user can capture all project requirements.
