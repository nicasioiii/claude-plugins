# Start Project

Help launch a new design project by understanding project goals, constraints, and complexity. This guides you through discovery and scoping using a structured framework.

---

## What This Command Does

You'll answer questions to:
- Define project goals and success criteria
- Assess target audience and user needs
- Determine project complexity (high/medium/low)
- Identify constraints (timeline, budget, technical)
- Create a project scope document
- Plan next steps in the design workflow

**Estimated Time:** 10-15 minutes

---

## Discovery Questions

I'll ask you a series of questions to build your project brief. Answer as detailed as you'd like:

**QUESTION 1: Project Overview**
What is the core goal of this project? What problem are you solving?

*Examples: "Building a SaaS dashboard for fitness tracking," "Redesigning our e-commerce store," "Creating a personal portfolio site"*

---

**QUESTION 2: Target Audience**
Who is the primary user for this project? Describe them in 2-3 sentences (role, goals, experience level).

*Examples: "Fitness enthusiasts aged 25-40 who want to track workouts," "Fashion shoppers looking for sustainable clothing," "Hiring managers seeking qualified candidates"*

---

**QUESTION 3: Key Features**
What are the 5-7 core features or sections this project needs?

*Examples:
- SaaS: Dashboard, user profile, settings, reporting, analytics, billing
- E-commerce: Product catalog, shopping cart, checkout, account, reviews
- Website: Hero, features, pricing, testimonials, contact, blog*

---

**QUESTION 4: Existing Reference**
Do you have any existing design, competitor sites, or style inspiration? Share links or descriptions.

*Examples: "Check out Notion's clean interface," "Our current site is at example.com," "Inspired by Stripe's minimalist aesthetic"*

---

**QUESTION 5: Constraints**
What are your timeline, budget, and technical constraints?

*Examples: "3 months, $15k budget, must work with Shopify," "2 weeks, internal team only, needs to be mobile-responsive"*

---

## Based On Your Answers

Once you've answered the questions, I'll provide:

### ✓ Project Complexity Assessment
**Low** (simple marketing website, few pages, standard patterns)
**Medium** (web app, custom components, some interactions)
**High** (complex SaaS, data visualization, many features, custom design)

**Why this matters:** Complexity determines how detailed your design process should be.

### ✓ Recommended Workflow
The skills you'll need in order:
1. **visual-foundations** — Establish design rules
2. **figma-workflows** — Organize your design file
3. **ui-components** — Design pieces
4. **web-layout** OR **dashboard-systems** — Assemble layout
5. **animation-interactions** — Add polish
6. **web-design-audit** — Review quality
7. **webflow-implementation** OR **shopify-design** — Build

### ✓ Discovery Checklist
Key items to clarify before starting design:
- [ ] Project goals and success metrics defined
- [ ] Target audience clearly described
- [ ] Core features and pages identified
- [ ] Style direction or inspiration gathered
- [ ] Stakeholder alignment on scope
- [ ] Timeline and resources confirmed
- [ ] Technical requirements documented
- [ ] Success criteria defined

### ✓ Scoping Matrix
|  | Low | Medium | High |
|---|-----|--------|------|
| **Pages/Screens** | 5-10 | 10-20 | 20+ |
| **Unique Components** | 3-5 | 8-12 | 15+ |
| **Interactions** | Basic hover/click | Custom animations | Complex interactions |
| **Timeline** | 1-2 weeks design | 2-4 weeks design | 4-8 weeks design |
| **Design System Needed?** | Simple | Yes | Comprehensive |

## Discovery Framework Reference

This command uses the **8-step design process** (Shift Nudge methodology):

1. **Discover** ← You are here
2. **Define** (create problem statement)
3. **Direct** (choose visual direction)
4. **Design** (create layouts and components)
5. **Develop** (build in design tool)
6. **Detail** (refine and polish)
7. **Deliver** (present to client)
8. **Deploy** (handoff to development)

You've completed step 1. Ready to move to step 2: define the design problem.

---

## Pro Tips

**Gather inspiration first:** Before design starts, collect 10-15 reference sites or designs that inspire you. Note what specifically appeals to you (colors? typography? layout?).

**Define constraints carefully:** "We have 2 weeks" is different from "We need to launch in 2 weeks." The scope must fit the timeline.

**Align stakeholders early:** Get everyone to agree on goals and constraints NOW, not mid-design. This prevents scope creep.

**Document decisions:** Keep notes on why you made certain choices. You'll need this for the hand-off to development.
