# Complexity Assessment: Determining Your Workflow

The complexity of your project determines:
- Whether you need wireframes
- How much discovery time to allocate
- Your design timeline and budget

Assess complexity early—it drives all downstream decisions.

---

## The Three Complexity Levels

### Low Complexity Projects

**Characteristics:**
- Simple, agreed-upon functionality
- Few pages (< 10)
- Linear workflows with no branching
- All stakeholders aligned on requirements
- Clear user goals with no ambiguity
- No special technical requirements
- Straightforward content structure

**Examples:**
- Landing page
- Marketing website (5-7 pages)
- Portfolio site
- Simple product page
- Blog or news site
- About/Contact pages

**Workflow:**
1. Minimal discovery (30 min call confirms everyone agrees)
2. Skip formal wireframes (everyone understands what's needed)
3. Create visual designs directly
4. Iterate based on feedback
5. Finalize and handoff

**Timeline:** 1-2 weeks

**Budget consideration:** Least expensive—straightforward design work

**Wireframing decision:** Optional. If you need to sketch to think clearly, sketch. But don't spend hours on formal wireframes.

**Discovery time:** 2-4 hours

---

### Medium Complexity Projects

**Characteristics:**
- Clear main use case but multiple features
- Multiple pages/sections (10-30)
- Some user flow uncertainty ("does feature A come before B?")
- Multiple user types with similar needs
- Some design system decisions needed
- Standard technical requirements
- Moderate stakeholder alignment needed

**Examples:**
- E-commerce site (product catalog, shopping cart, checkout)
- Content marketing site (hero, features, testimonials, blog)
- Simple SaaS app (dashboard, settings, team management)
- Restaurant/service website (menu, reservations, about, contact)
- Startup website (product, pricing, blog, team)

**Workflow:**
1. Discovery call to understand goals and audience
2. Create sitemap and key user flows
3. Create wireframes (medium-fidelity: sizing, spacing, content placement)
4. Get stakeholder feedback on wireframes (before visual design)
5. Create visual designs with colors, typography, images
6. Iterate on visual designs based on feedback
7. Finalize and handoff

**Timeline:** 3-6 weeks

**Budget consideration:** Medium—requires wireframes + visual design

**Wireframing decision:** REQUIRED. Wireframes separate functional conversations from visual conversations. Saves time overall.

**Discovery time:** 6-10 hours

**Wireframe time:** 4-8 hours (high-fidelity wireframes with typography, spacing, real content)

---

### High Complexity Projects

**Characteristics:**
- Unclear or contested requirements
- Multiple user types with conflicting needs
- Complex workflows with many branching paths
- Regulatory or accessibility mandates
- Large feature set (30+ pages)
- Stakeholder misalignment
- Novel interaction patterns
- Significant technical constraints
- Data-heavy or interaction-heavy interfaces

**Examples:**
- Banking app or fintech product
- Medical/healthcare platform
- Government portal
- Complex SaaS dashboard
- Marketplace platform
- Admin/CMS interface
- Real estate or logistics platform

**Workflow:**
1. Extended discovery (1-2 hours on call + async research)
2. User research (interviews, personas, mental models)
3. Create sitemap and detailed user flows
4. Detailed wireframes with annotations (30min-1hr per page)
5. Prototype key flows in Figma
6. Stakeholder walkthrough of flows
7. Create visual designs after flows approved
8. Usability testing (if budget allows)
9. Iterate on both interactions and visuals
10. Finalize and handoff

**Timeline:** 6-12 weeks

**Budget consideration:** Most expensive—extensive discovery, wireframes, prototypes, testing

**Wireframing decision:** ESSENTIAL. Prototyping also recommended to validate interactions.

**Discovery time:** 10-20 hours

**Wireframe time:** 12-20 hours (very detailed wireframes with annotations on every interaction)

## Quick Assessment Questions

Answer these to determine complexity:

**1. Stakeholder alignment:**
- Everyone agrees on requirements? → Lower complexity
- Stakeholders have different visions? → Higher complexity

**2. Feature clarity:**
- All features clearly understood? → Lower complexity
- Any unclear features or dependencies? → Higher complexity

**3. User types:**
- Single user type with same goals? → Lower complexity
- Multiple user types (admin vs. user vs. guest)? → Higher complexity

**4. Workflows:**
- Linear, straightforward paths? → Lower complexity
- Complex branching, conditional logic? → Higher complexity

**5. Content structure:**
- Flat, simple organization? → Lower complexity
- Nested hierarchies, many data relationships? → Higher complexity

**6. Technical requirements:**
- Standard web requirements? → Lower complexity
- APIs, integrations, custom backend? → Higher complexity

**7. Constraints:**
- No special requirements? → Lower complexity
- Accessibility mandates, security/compliance needs? → Higher complexity

## Complexity Flowchart

```
START: New Project
  ↓
Are all stakeholders aligned on requirements?
├─ NO → High Complexity (skip to High path)
└─ YES → Continue
      ↓
Are there more than 3 distinct user types?
├─ YES → Likely Medium or High
└─ NO → Continue
      ↓
Are the main workflows straightforward and linear?
├─ NO → Medium or High Complexity
└─ YES → Continue
      ↓
Are there special requirements (compliance, APIs, custom tech)?
├─ YES → Medium or High Complexity
└─ NO → Continue
      ↓
Is feature count < 10 pages?
├─ YES → LOW COMPLEXITY
│   Do: Minimal discovery, skip wireframes, design directly
│   Timeline: 1-2 weeks
│
└─ NO (10-30 pages) → MEDIUM COMPLEXITY
    Do: Discovery call, sitemap, wireframes, visual design
    Timeline: 3-6 weeks

If any NO answers above → HIGH COMPLEXITY
Do: Extended discovery, user research, wireframes, prototype, test
Timeline: 6-12 weeks
```

## Complexity Reassessment

You might start as "low" and discover it's actually "medium" or "high" during discovery. That's okay—reassess after:

- Initial client call
- After researching competitors
- After mapping workflows
- After stakeholder alignment meeting

**If complexity increases:**
- Adjust timeline
- Increase wireframing effort
- Request additional budget
- Add prototype phase

**If complexity decreases:**
- Reduce wireframing effort
- Accelerate visual design
- Reduce timeline

## Wireframing Effort Guide

### Low Complexity: Optional Wireframes

If you wireframe, spend 0-4 hours:
- Sketches on paper or Figma rough shapes
- Key content areas labeled
- Basic flow documented
- Focus: Clarify your own thinking, not for client review

### Medium Complexity: High-Fidelity Wireframes

Budget 4-8 hours:
- Figma wireframes with actual sizing and spacing
- Real content or realistic placeholders
- Clear typography hierarchy (sizes, weights)
- Navigation structure shown
- Key interactions annotated
- 1-2 hours per major page

**Deliverable quality:** Client-ready for feedback

### High Complexity: Detailed + Prototype

Budget 12-20 hours for wireframes:
- Detailed wireframes (as above)
- Full annotation of interactions
- Multiple user flows with screenshots
- Prototype in Figma for key workflows
- 30min-1hr per page for detailed annotation
- Prototype: 4-8 hours for key flows

**Deliverable quality:** Ready for usability testing

## Cost-Benefit of Wireframing

### The Wireframe Investment Calculus

**Low Complexity:**
- Wireframe time: 0-4 hrs
- Rework if skipped: 0-5 hrs
- **Verdict:** Optional; wireframes only if uncertain

**Medium Complexity:**
- Wireframe time: 4-8 hrs
- Rework without wireframes: 15-30 hrs
- **Savings:** 7-22 hrs
- **Verdict:** ALWAYS wireframe for medium complexity

**High Complexity:**
- Wireframe + prototype time: 16-28 hrs
- Rework without proper wireframes: 40-80 hrs
- **Savings:** 12-64 hrs
- **Verdict:** ALWAYS wireframe + prototype for high complexity

**Key insight:** The more complex the project, the more wireframing saves you overall.

## Communicating Complexity to Clients

When presenting your assessment:

**For Low Complexity:**
"Your project is straightforward—we can move directly to design. The discovery call confirmed everyone agrees on requirements and features."

**For Medium Complexity:**
"I recommend creating wireframes first. This separates functional conversations (does this feature work?) from visual conversations (colors, style). It's more efficient and prevents surprises."

**For High Complexity:**
"Given the complexity, I recommend investing in wireframes and a prototype for key workflows. This ensures we're solving the right problem before committing to a design direction. Testing with users would validate our assumptions."

## Red Flags: Complexity Creep

Watch for these signals that complexity is increasing mid-project:

1. **New features appearing:** "Oh, we also need..."
   - **Action:** Document and reassess timeline/budget

2. **Stakeholder disagreement:** Conflicting feedback on direction
   - **Action:** Hold alignment meeting before continuing

3. **Technical surprises:** "We need to integrate with X API"
   - **Action:** Understand API constraints before design

4. **User research revealing gaps:** Interviews uncover missing use cases
   - **Action:** Extend discovery timeline

5. **Unclear success metrics:** Goals changing each week
   - **Action:** Pin down metrics before design finalization

## Next Steps

Once complexity is determined:

1. **Low:** Move directly to visual design
   - References: Use `visual-foundations` + `figma-workflows`

2. **Medium:** Create wireframes, then visual design
   - First: Wireframe key pages (3-5 hours)
   - Then: Move to `visual-foundations` + `figma-workflows` + `ui-components`

3. **High:** Extended discovery + wireframes + prototype + design
   - First: Complete extended discovery (10-20 hours)
   - Then: Create detailed wireframes (12-20 hours)
   - Then: Prototype key flows (4-8 hours)
   - Finally: Visual design phase
