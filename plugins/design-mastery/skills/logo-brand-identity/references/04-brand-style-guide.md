# Brand Style Guide: Structure, Clear Space, Export, and Presentation

Complete style guide template, clear space rules, file export protocol, and client presentation strategy.

---

## STYLE GUIDE STRUCTURE [LogoCore]

### Purpose

Without guidelines, clients will distort logos with Instagram filters, crowd layouts, print on bad printers with color shift. Style guides prevent brand deterioration as more people use the logo.

**Approach it as a book/story, not random graphics.** Once you create one style guide you love, reuse that template across clients.

### Minimum Sections

| Section | Content |
|---------|---------|
| 1. Introduction/cover | Brand story, identity overview |
| 2. Primary logo display | Full logo at generous size |
| 3. Logo variations | With type, without type, icon-only, square format |
| 4. Logo misuse rules | How NOT to use the logo (stretch, rotate, recolor) |
| 5. Color palette | HEX, RGB, CMYK, Pantone values for every brand color |
| 6. Typography | Primary + secondary typefaces, weights, usage rules |
| 7. Dark/light versions | Logo on dark backgrounds, logo on light backgrounds |

### Extended Sections (Budget Permitting)

- Photography style guidelines
- Stationery design examples (business card, letterhead, envelope)
- Social media application examples
- Icon system guidelines
- Pattern/texture library
- Signage and environmental applications
- Digital advertising specifications
- Email signature format

## CLEAR SPACE RULES [Will Patterson]

### Defining Clear Space

1. Measure using a proportion of the logo itself (e.g., x-height of the logotype)
2. Use that measurement as minimum buffer on all sides
3. No text or graphic should enter the clear space area
4. Create clear space guides with Cmd+5 (Convert to Guides)

### Baking Clear Space into Export Files

1. Create an artboard that includes the clear space area
2. Export SVG/PNG with clear space already in the bounding box
3. When client uses the file, clear space is automatically maintained
4. Prevents clients from placing elements too close to the logo

## FILE EXPORT PROTOCOL [LogoCore]

### Export Checklist Per Logo Variation

| Format | Specifications |
|--------|---------------|
| PNG | 1x, 2x, 4x, 8x (transparent background) |
| JPG | 1x, 2x, 4x, 8x (white background) at quality 100 |
| SVG | Start with SVG Tiny 1.2; if breaks, try Basic 1.1, then 1.1, then 1.0 |
| AI | Save as CS6 for compatibility; enable "Generate PDF Compatible File" |

**File naming:** Prefix all files with company name (e.g., "skiworld-logo-primary-color.svg")

### SVG Optimization [LogoCore]

- Target smallest possible file size (example: 1.3KB for logo vs 29KB for equivalent JPG)
- SVG renders crisp at any zoom level, any device, any retina display
- Verify by dragging SVG into Chrome to test rendering

### Variations to Provide

- Primary logo (full color)
- Primary logo (single color / outline)
- Logo + type lockup
- Icon-only version
- All above in white (for dark backgrounds)
- Favicon at 64x64px

### Logo Package Express [Will Patterson]

- Paid Illustrator extension (~$100+) that automates entire export process
- Generates all format variations, print/web folders, color worksheet with Pantone values
- Don't give clients file types they won't use (e.g., no print version for web-only clients)

### Logo Package Swatch [Will Patterson]

- Extracts Pantone, CMYK, RGB, HEX values automatically from selected artwork
- Generates color worksheets for the style guide

## CLIENT PRESENTATION STRATEGY

### Five Keys to a Winning Presentation [Will Patterson]

1. **Be prepared:** Know key talking points; have mockups ready; understand both strengths AND weaknesses; have a plan to mitigate client fears
2. **Be yourself:** Don't imitate other designers' presentation styles; be kind and courteous
3. **Be authentic:** If you don't like part of the design, say so. [CONTRARIAN] Don't manipulate clients with fake grids or overstated reasoning
4. **Show off:** Include process images, behind-the-scenes sketches, overhead camera shots of sketchbooks. The client buys your creative mind, not just the logo
5. **Meet face to face:** Video call minimum; read body language. If impossible, screen-record yourself presenting with webcam visible

### Presentation Construction [LogoCore]

**Mock-up usage:**
- Place logo in realistic context: business cards, website headers, app icons, signage
- Use Photoshop mockups from sites like GraphicBurger
- Smart Objects allow quick logo swapping across multiple scenes

**Behance case study structure:**
- Plan as blocks: sketch process, kitbashing, final logo, variations, mockups
- Consistent background color throughout
- Show creative process, not just final result

### Framing Client Feedback

**Ask specific questions:**
- "Which direction resonates with your brand?"
- "Where could we take this direction further?"
- "What's not working about this concept?"
- Do NOT ask vague "What do you think?" -- get specific feedback

### Revision Process

**Standard scope:** 2-3 revision rounds included in project scope

| Round | Purpose |
|-------|---------|
| Round 1 | Major direction changes (client picks preferred direction) |
| Round 2 | Refinements and adjustments |
| Round 3 | Final tweaks and polish |

### Handling Difficult Feedback

| Client Says | What They Mean | How to Respond |
|-------------|---------------|----------------|
| "Make it more creative" | They can't articulate the issue | Ask for specifics: what elements feel uncreative? |
| "I don't like the color" | Color doesn't match their vision | Test alternatives; ask which direction they prefer |
| "Make it bigger/smaller" | Usually complexity or concept issue | Explain that size isn't usually the problem |
| "Can we combine concept A and B?" | They like elements of both | Evaluate if combination works; often it doesn't |
| "My wife/friend doesn't like it" | Outside opinions introduced | Redirect to brand strategy: does it serve the audience? |

## LOGO ANIMATION PRINCIPLES [LogoCore]

Core principles for logo reveals (After Effects excluded from scope; principles only):

| Principle | Description |
|-----------|-------------|
| Eases | Never use linear keyframes; always ease-in/out for natural motion |
| Arcs | Objects follow curved paths, not straight lines |
| Anticipation | Small preparatory movement in opposite direction before main action |
| Follow-through | Elements continue slightly past endpoint before settling |
| Squash and stretch | Compress on impact, elongate during fast movement |

**Duration:** Keep animations short (2-5 seconds for logo reveals).

## PRICING REFERENCE [Will Patterson]

| Service Level | Starting Price | Includes |
|---------------|---------------|----------|
| Logo + mini brand guidelines | ~$10,000 | Logo suite, basic style guide, color/type specs |
| Full brand identity | Higher (bespoke) | Complete system, stationery, applications |
| Entry-level (building portfolio) | $500-$1,000 | Start here as a challenge; raise incrementally |

**Value-based pricing:** Charge based on value to the client, not hours spent. Nike pays more than a local coffee shop because Nike gains more value.

**Specialization premium:** "It's not how, it's who" -- niching down (e.g., "brand identity designer for luxury brands in Dubai") enables premium pricing.

**Full price or free philosophy:** Never discount. Penalizes early buyers.
