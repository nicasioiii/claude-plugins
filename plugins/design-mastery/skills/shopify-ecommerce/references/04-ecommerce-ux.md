# E-Commerce UX & Conversion

Conversion-optimized design patterns for Shopify stores.

---

## CONVERSION FUNNEL PRINCIPLES

Convert browsers into buyers through five principles:

1. **Emotional** -- convey lifestyle, not just product shots. Brands are remembered for feelings
2. **Clear** -- effects should not distract from the product. Customers need to see products clearly
3. **Fast** -- heavy effects slow load times. Consider mobile users on poor connections
4. **Easy** -- don't hide content or distract. Seamless navigation encourages repeat purchases
5. **Scannable** -- users scan pages, not read them. Use visuals for instant impressions. Key info in headings for scanners

## HOMEPAGE STRUCTURE (CONVERSION-OPTIMIZED)

### Above the Fold (8-Second Attention Span)

**1. Announcement Bar**
- Showcase special offers (10% off, free shipping over $X)
- Always add placeholder text if client hasn't provided copy

**2. Header/Navigation**
- No hamburger menus on desktop -- they hide information
- Maximum 7 navigation links
- Items should be shopping-related (collection titles that communicate what's sold)
- No FAQ or Home links needed (logo = home)
- Logo wider than tall; square logos waste vertical space
- Remove logo taglines -- hero section explains the brand
- Header: maximum 15% of viewport height

**3. Hero Section**
- Strong descriptive image + catchy headline (include keywords for SEO)
- CTA longer than just "Shop" -- e.g., "Discover high-quality hemp oil"
- Show product AND people (emotion + product)
- Position people looking/turning toward product (draws the eye)
- [CONTRARIAN] No sliders -- proven to hide content; shoppers don't see most slides
- Switch hero content often -- treat website like a storefront (seasonal, launches)

### Below the Fold (Priority Order)

**4. Press Features**
- Publication logos, maximum 8
- Place high on page for credibility

**5. Featured/Best-Selling Products**
- Descriptive headings: "Shop Ethical Jewelry" not "Featured Products"
- Place high on page

**6. Product Grid Best Practices**
- Must include: product title, reviews, price (design for sale prices too)
- Image rollover: lifestyle/model image on hover
- Buy now / add to cart buttons (reduces clicks for impulse products)
- Quick view: good for products with variants (avoid PDP navigation)
- Extra info line: secondary title if product name doesn't self-describe
- Product badges: sale, low stock, best seller, custom ("Featured in Vogue")
- Color swatches on grid items

**7. Social Proof / Reviews**
- Front and center
- Include UGC (user-generated content) showing product in use

**8. Brand Values / USPs**
- Icons + short text for scannability
- Break information into visual chunks

**9. Hero Product / Bundle**
- Feature prominently
- Lets shopper be sold without clicking to PDP

**10. About Section** (optional)
- Brand phrase as heading, not generic "About Us"
- Include CTA to read more or shop
- Only if client has limited homepage content or unique story

**11. Blog Summary** (optional)
- Only if blog is important; mainly for SEO

**12. Newsletter Signup**
- Usually in footer to save space
- Encourage offer in exchange for email
- Stress: email list is only audience channel they truly own

**13. Instagram Feed** (optional)
- Only if client has strong Instagram
- Needs refreshing every few months

**14. Footer**
- Acts as site map
- Alternative logo + one-sentence brand description
- Social links
- Policies, copyright, terms
- Payment gateway icons (only if required)

### Homepage Length Rule
- Don't make it too long -- sections stack on mobile, becomes even longer
- Create toggle-able sections so clients can switch on/off for fresh content
- [CONTRARIAN] Always include your credit at the bottom next to copyright

## PRODUCT DETAIL PAGE (PDP)

### PDP Importance
- Shoppers may land directly from Google -- PDP must convert independently
- PDP is as important as homepage

### Required Elements

| # | Element | Notes |
|---|---------|-------|
| 1 | Product images | Display method depends on theme; get preference early |
| 2 | Product title | Must be H1 tag for SEO |
| 3 | Secondary title | Optional additional context |
| 4 | Review summary | Social proof at point of purchase |
| 5 | Variants | Size, color selectors + optional size guide |
| 6 | Quantity selector | Only if shoppers might buy multiples (bath soaks YES, jewelry NO) |
| 7 | Add to cart | Contrasting fill color, highly visible |
| 8 | Buy now | Optional; filled style (add to cart gets outline). For single-item purchases |
| 9 | Product description | Snappy, include info not visible in photos |
| 10 | Tabs/collapsible rows | Ingredients, how to use, shipping info |
| 11 | Extended content | Optional for small catalogs or hero products as sales page |
| 12 | Reviews section | Full reviews below the fold |
| 13 | Recommended products | Prevents dead-end pages; introduces products to Google visitors |

### PDP Design Best Practices
- Follow standard layout (images left, info right) -- shoppers expect this
- Gray text for secondary info (creates hierarchy)
- "Read more" links: bold AND underlined (clearly clickable)
- Icons for product features (scannable at a glance)
- Two columns of product images (front + model) to see both without scrolling
- One alternative font color for information hierarchy
- Include breadcrumbs for navigation context

## MOBILE DESIGN RULES

### Typography
- Body copy: minimum 14px
- Form input text: minimum 16px (prevents iPhone auto-zoom on focus)

### Navigation
- Sticky header is fine on modern tall screens
- Consider bottom-of-screen navigation (easier thumb access)

### Layout
- Product grid: always 2 columns on mobile (1 column = too much scrolling)
- Buttons: large enough for thumbs with adequate spacing
- Remove decorative floating stickers if they obstruct content
- Sliders useful for space conservation (make sliding obvious: edge fade or arrows)

### Custom Sections
- Plan mobile stacking in advance
- Even-number columns work better for 2-column mobile grid
- 5 columns on desktop becomes problematic on mobile

### Mobile Pop-ups
- Prefer corner-sitting over full-screen interruptions
- Phone number collection gaining popularity (better engagement for younger audiences)

## SEO CHECKLIST FOR SHOPIFY

### Before Launch
- [ ] Meta descriptions: homepage (most important), all pages, products, blog posts
- [ ] URL handles: verify ALL (no "copy-of-" or "-1" artifacts)
- [ ] Image alt text: ALL images (product images in product editor, others in Settings > Media)
- [ ] Image file names: rename descriptively BEFORE uploading
- [ ] HTML headings: one H1 per page, proper hierarchy
- [ ] Blog posts: suggest to clients (authority, long-tail keywords, freshness signal)
- [ ] Sitemap: submit to Google Search Console

### Ongoing
- [ ] Blog updates for freshness signal
- [ ] Product descriptions with relevant keywords
- [ ] New product images with alt text
- [ ] URL structure maintained (no breaking changes)
