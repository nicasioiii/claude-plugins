# Hierarchy Checklist & Callout Techniques

## HIERARCHY CHECKLIST [MDS]

Before finalizing any screen's typography, answer these four questions:

1. **What do you think the user is planning to do on this screen?**
2. **Are they browsing/exploring or looking for something specific?**
3. **What are you hoping the user does/can do?**
4. **What is the #1, #2, #3 most important thing on screen? = hierarchy**

### Key Principles

- Titles do not have to be larger just because they are titles
- An H2 does not need to be the second-largest thing on screen -- context matters more than semantic level
- Business needs can drive hierarchy (e.g., "Set up auto pay" module gets 2x title size because it reduces support headaches)
- More negative space around an element = more power/gravitas
- White background with dark text has less visual weight than dark background with white text (same size)

### Seven Tools for Hierarchy [Flux]

1. **Order** -- top-left reads first (Latin languages)
2. **Spacing/isolation** -- one item alone among groups gains prominence
3. **Alignment shift** -- moving off shared alignment makes it stand out
4. **Value/weight** -- bold vs. regular, opacity changes (go up at least 2 weight steps)
5. **Color** -- a single red among grays commands attention
6. **Scale** -- dramatically bigger, not slightly bigger
7. **Enclosure/notation** -- strokes, rules, stars, containers

**Critical mistake:** Making variations too subtle. Hierarchy requires dramatic contrast, not incremental difference. [Flux]

## CALLOUT TECHNIQUES [MDS]

### Five Methods

1. **Size** -- make it larger (32pt+ pull quotes vs. 18pt body)
2. **Case** -- uppercase adds weight; allows tighter letter-spacing
3. **Style/Color** -- bold, italic, or colored text; Georgia italic is especially elegant for quotes
4. **Lock-up elements** -- combine with quotation marks, logos, avatars, borders, background colors
5. **Breaking the format** -- offset quotes left/right of body column; break the grid

### Pull Quote Formula

Larger typeface + bold weight + quotation mark graphic + background color + avatar + name lock-up = maximum callout impact.

As importance elevates, increase: negative space, bold colors, combination with images and decorative elements.

### Long-Form Page Callouts

Within long-form pages (3000-5000 words), use 4-5 different callout element styles to maintain visual interest throughout scrolling.

### Block Quote Accent

A thin 1-4px colored vertical line (brand accent color) as left border = simple, elegant block quote indicator.

## INTERACTIVE TEXT COLOR SYSTEM [MDS]

### Color Rules

1. **Primary interactive color:** Brand color or blue for main actions
2. **Secondary interactive color:** Lighter gray for less-important interactive elements
3. **Danger/destructive color:** Red for delete and dangerous actions
4. **Never overuse the interactive color** -- if too many things are blue, the signal weakens

### Position-Based Interactivity

- Navigation items at the top do not need color to look clickable -- position alone signals interactivity (learned pattern)
- Weight for active state: bold weight on sidebar nav items indicates current/active page

### Input Field Interactivity

- Inputs look interactive because of their border/background, not text color
- Placeholder text should be lighter than body text -- NOT the same color as interactive elements
- Ask "why" when requesting personal info (e.g., phone number) -- explain the benefit

### Accessibility Rule (WCAG)

Color alone should NOT be the only way to convey information/action. Combine with underline, weight, position, or icons.

## TYPOGRAPHIC TEXTURE & RHYTHM [Gareis]

### Texture (Typographic "Color")

- **Lighter texture** (lighter weight + increased line spacing + more paragraph spacing) = airy, open feeling = luxury brands, contemporary designs, approachable content
- **Denser texture** (heavier weights + tighter line spacing + compact paragraphs) = substantial, serious = academic content, authority-establishing designs

### Rhythm

- Created through consistent spacing relationships, repeated elements, and deliberate interruptions
- Consistent paragraph indentations create a regular pattern guiding the reader
- Breaking established rhythm patterns creates emphasis through contrast
- Varying alignments and weights create focal points without losing coherence

## CREATING EMPHASIS WITHIN TEXT [Gareis]

### Five Techniques

1. **Contrasting font weights/styles** -- pair thin serif uppercase with flowing italic script
2. **Bold color contrast against background** -- vibrant color captures attention without changing letterform
3. **Size and scale variation** -- dramatically enlarge key element while keeping context text small
4. **Environmental context and white space** -- generous negative space ensures text commands attention
5. **Mixing font styles with decorative elements** -- different words receive different typography treatments

**Consistency rule:** Whichever emphasis technique you choose, apply it CONSISTENTLY throughout the entire design project. Inconsistency confuses viewers.

---

## MAINTAINING TYPOGRAPHIC HARMONY [Gareis]

### Five Rules

1. **Limit typefaces** -- max 2-3 font families with selected weights for entire design
2. **Consistent alignment patterns** -- intentional exceptions draw attention to key messages
3. **Consistent spatial relationships** -- same margins/spacing proportions throughout
4. **Established paragraph formatting** -- if using indentation, use throughout; never mix methods
5. **Stylistic consistency in special treatments** -- if using small caps for section headings, use for ALL section headings

## COMMON TYPOGRAPHY MISTAKES [MDS Critique Vault]

### From Contact Card Reviews (73 reviews)
- Too many font sizes (breaking Rule of Four)
- Inconsistent spacing/padding between elements
- Poor hierarchy -- everything competing at the same level
- Colors not passing accessibility contrast
- Truncation not accounted for with long names

### From Font Size Reviews (66 reviews)
- Too many different sizes
- Inconsistent line-heights across same-size text
- Title sizes too similar to body (need more contrast)
- Metadata too large (should be clearly subordinate)

### From Font Weight Reviews (28 reviews)
- Over-reliance on bold -- everything bold = nothing bold
- Not enough weight contrast between hierarchy levels
- Light weights at small sizes (unreadable)
- Missing opportunity to use weight for interactive signaling

### From Hierarchy Reviews (48 reviews)
- Flat hierarchy -- all elements at same visual level
- Over-designing lower-priority elements
- Not considering what user wants to accomplish on each screen
