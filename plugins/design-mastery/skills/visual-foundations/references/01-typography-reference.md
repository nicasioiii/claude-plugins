# Typography Reference: Complete Sizing & Rules

Typography is your most powerful design tool. These exact specifications come from production design work across web, iOS, Android, and marketing sites.

---

## Font Size Standards by Context

### Web Design (Desktop)

**For 16px browser base (1em = 16px):**

| Role | Size | Weight | Line Height | Use |
|------|------|--------|-------------|-----|
| H1 | 48px | 600-700 | 120% | Page headlines, hero text |
| H2 | 32px | 600-700 | 120% | Section headers |
| H3 | 24px | 600 | 120% | Subsection headers |
| Body | 16px | 400 | 150% | Main content, paragraphs |
| Small | 14px | 400 | 150% | Secondary text, captions |
| Tiny | 13px | 400 | 150% | Metadata, timestamps, footnotes |

**Alternative safe scale:** 40px / 28px / 20px / 16px / 14px / 13px

**Another option:** 56px / 36px / 20px / 16px / 14px / 13px

**Key rule:** Whatever scale you choose, stick with it. Don't add a 5th or 6th size.

### Mobile Applications (iOS)

**iOS-standard sizing:**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Jumbo title | 34px | 600-700 | 120% |
| Title | 20px | 600-700 | 120% |
| Headline | 17px | 600-700 | 120% |
| Body | 16px | 400 | 150% |
| Subtext | 15px | 400 | 150% |
| Small | 13px | 400 | 150% |

**Key:** iOS title is 17px standard (not 18px). Typography baseline aligns with 16px body.

### Mobile Applications (Android)

**Material Design standards:**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display large | 57px | 400 | 120% |
| Display medium | 45px | 400 | 120% |
| Display small | 36px | 400 | 120% |
| Headline large | 32px | 700 | 120% |
| Headline medium | 28px | 700 | 120% |
| Headline small | 24px | 700 | 120% |
| Title large | 22px | 700 | 120% |
| Title medium | 16px | 700 | 120% |
| Title small | 14px | 700 | 120% |
| Body large | 16px | 400 | 150% |
| Body medium | 14px | 400 | 150% |
| Body small | 12px | 400 | 150% |
| Label large | 14px | 500 | 120% |
| Label medium | 12px | 500 | 120% |
| Label small | 11px | 500 | 120% |

**Key:** Material Design separates display vs. headline vs. title vs. body. More granular than iOS.

### Marketing/Landing Pages

**Bigger, bolder approach:**

| Role | Size | Weight |
|------|------|--------|
| Hero headline | 64-96px | 700 |
| Section headline | 48px | 700 |
| Subsection | 32px | 600 |
| Body | 18px | 400 |
| Small | 14px | 400 |

**Key:** Marketing copy is larger overall. Hero headlines are bigger and bolder.

## Font Weight Guide

### Available Weights

Not all fonts include all weights. **Check before using.**

**Standard availability (most fonts):**
- 400 (Regular) ✓ Always available
- 700 (Bold) ✓ Usually available
- 300 (Light) - Check before using
- 600 (Semi-bold) - Often available

**Extended availability (premium fonts):**
- 100 (Hairline) - Rare, use only 60px+
- 200 (Thin) - Rare, use only 60px+
- 500 (Medium) - Less common
- 800 (Extra bold) - Less common
- 900 (Black) - Rare, use sparingly

### Weight Application Rules

**Light/Hairline (100-200):**
- ❌ NEVER use for body copy (16-18px). Unreadable, strains eyes.
- ✓ OK for very large display text (60px+)
- ✓ OK for short snippets at large sizes only
- Example: 96px hero headline in light weight works

**Regular/Book (400):**
- ✓ Standard for body copy
- ✓ Can use same weight as titles with color/size to differentiate
- Standard choice for readability

**Semi-Bold/Medium (500-600):**
- ✓ Headlines and titles
- ✓ Creates hierarchy when combined with 400 body
- ✓ Better than jumping straight to 700 (more refined)

**Bold/Heavy (700-900):**
- ✓ Primary headlines
- ✓ CTAs and prominent buttons
- ✓ Creates strong visual emphasis
- ❌ Don't use for body copy (too heavy, hard to read)

### Visual Weight vs. Font Weight

Weight is only part of hierarchy. These also add visual weight:

**Negative space:** Large white space around text = heavier visual weight

**Color:** Darker colors = heavier visual weight

**Size:** Larger = heavier visual weight

**ALL CAPS:** More weight than title case (can use tight line height)

**Background:** Dark background + light text = heavier than light text alone

## Line Height: Readability Science

### Body Copy Line Height

**Standard:** 150% (1.5x) for optimal readability

**Range:** 125%-220% depending on typeface and context

**Calculation:** Font size × line height = space between lines
- 16px × 150% = 24px between baselines

### Why Line Height Matters

**Too tight (<125%):**
- Lines feel cramped
- Eyes hard to track left-to-right
- Reader fatigue
- Example: 16px text with 100% line height = hard to read

**Too loose (>200%):**
- Content feels disconnected
- Hard to scan related paragraphs
- Loss of rhythm
- Example: 16px text with 200% line height = feels fragmented

**Sweet spot (150%):**
- Natural rhythm
- Easy to follow line-to-line
- Comfortable for extended reading
- Works for most contexts

### Headline Line Height

**Titles: 100-120%** (much tighter than body)
- Multi-line titles need different rules
- Watch for descenders (g, y) colliding with ascenders (A, K)

**All caps: 75-80%** (can be tighter due to blockier letters)
- All caps letters don't have descenders
- Tighter spacing creates intentional compactness

### Context Adjustments

**Long copy (articles, documentation):** 160-180% for maximum readability

**Short copy (UI labels, buttons):** 110-120% to keep compact

**Headlines (1-3 lines):** 100-120%

**Narrow columns (<45 characters):** 140-150% (tighter works in constrained spaces)

**Wide columns (>75 characters):** 160-180% (readers need help tracking across distance)

## Character Width: Optimal Reading Length

### The 45-75 Character Rule

**Optimal range:** 45-75 characters per line

**Calculation:** Count characters (letters, spaces, punctuation) per line

### Why Character Width Matters

**Too short (<45 characters):**
- Awkward line breaks
- Hard visual rhythm
- Fragmented reading flow
- Example: 30-character line = choppy

**Too long (>75 characters):**
- Eyes hard to track line-to-line
- Reader loses place
- Cognitive load increases
- Example: 100+ character line = straining

**Sweet spot (60 characters):**
- Natural rhythm
- Easy to track to next line
- Comfortable for extended reading

### How to Calculate

**Method 1 (simple):** Measure characters on your longest line

**Method 2 (CSS):**
```css
/* Approximate 60 characters at 16px */
max-width: 35em; /* 16px × 35 = 560px */

/* Approximate 65 characters at 18px */
max-width: 38em; /* 18px × 38 = 684px */
```

**Method 3 (practical):** Look at your design. Does the text feel comfortable to read? If yes, it's probably 45-75 characters.

### Context Variations

**Short form (UI, buttons, labels):** 20-40 characters acceptable

**Medium form (descriptions, metadata):** 40-60 characters ideal

**Long form (articles, documentation):** 50-75 characters ideal

## Truncation: Handling Overflow Content

### Truncation Rules

When content is too long, truncate with ellipsis (...) instead of wrapping.

**Single-line truncation:** Most common
- Titles in lists
- User names
- Blog post titles in feeds
- Technique: `text-overflow: ellipsis; white-space: nowrap;`

**Multi-line truncation:** 2-3 lines before ellipsis
- Description text
- Meta descriptions
- List item subtitles
- Shows preview of content

### Real-World Examples

**Podcasts App:**
- Podcast name: Single-line truncation (title only)
- Episode title: 3-line max, then ellipsis
- If short title, show description preview

**Twitter/Social:**
- Names/usernames: Truncate
- Tweet body: Full content visible (scroll if needed)
- Comments: Truncate if thread, show "More"

**E-commerce:**
- Product title: 1-2 line max, truncate
- Price: Always visible (core info)
- Description: 1 line or expand/see-more link

### Dynamic Content Solutions

**Truncate + expandable link:**
```
Long title here that's too long...
[See more]
```

**Truncate + tooltip (desktop):**
```
Long title truncated [hover to see full]
```

**Ingredient-based truncation:**
- List all content "ingredients" (price, rating, availability)
- Show subset on list view (most relevant)
- Link to expanded detail view
- Reduces cognitive load on scannable lists

## Letter Spacing: Refinement Detail

### Standard Letter Spacing

**Default (no adjustment):** Most fonts work fine with default spacing

**Tight letter spacing:** -2% to -4% for visual refinement
- Especially effective for geometric sans-serifs
- Example: Enter (Futura-like) with -3% to -4% = transforms ugly to beautiful
- Creates more refined, compressed appearance

**Loose letter spacing:** +2% to +5% for display text
- All caps headlines (more breathing)
- Small text on large backgrounds
- Decorative/display purposes

### When to Adjust

**Always test visually**, not by formula.

**Geometric sans-serifs benefit most:** -3% to -4% is standard for refinement

**Script/decorative fonts:** Often need -1% to -2% to look right

**Serifs:** Usually work fine with default

**All caps:** Often need -1% to -2% (letters are more rectangular, feels loose by default)

### Implementation

**In Figma:** Letter spacing is in typography panel (% or px)

**In CSS:**
```css
/* -2% letter spacing */
letter-spacing: -0.02em;

/* -4% letter spacing */
letter-spacing: -0.04em;
```

## Font Pairing Quick Guide

### Rules for Pairing

**Do's:**
- Limit to 2-3 fonts maximum (usually: 1 display + 1-2 body)
- Make pairings intentional and visually distinct
- One should be display/headline, other for readability
- Don't fear contrast—very different fonts can be complementary

**Don'ts:**
- Don't pair two similar-looking sans-serifs (looks indecisive)
- Don't use more than 3 fonts (appears chaotic)
- Don't pair two fonts that do the same job

### Common Pairings (from production)

**Bold Display + Rounded Friendly:**
- Russian (display) + PP New Montreal (body)
- Result: Angular boldness + rounded friendliness

**Playful Display + Approachable Body:**
- Maple (display) + Peppermint Wobry (body)
- Result: Playful + readable

**Single Font Solution:**
- Rubik (single font works when it's versatile)
- Works well if font has multiple weights

### Testing Pairings

- Test in your actual design context
- Don't rely on fonts.com previews alone
- What works in one layout may not work in another
- Spend time consuming beautiful design to train eye

```

  ↓ 120% line height

  ↓
```

## Accessibility & Typography

### Minimum Size for Readability

- **Body copy:** 16px minimum (web), 15px minimum (mobile)
- **UI labels:** 14px minimum
- **Small text:** 12px acceptable for timestamps, metadata

### Contrast Requirements

- **Body text:** 4.5:1 contrast minimum
- **Large text (18.5px+ bold):** 3.0:1 minimum
- **Small text:** 7.0:1 preferred

### Responsive Text Sizing

**Mobile:** Text may scale slightly smaller due to screen proximity
- Base body: 16px (web) → 15px (iOS) → 14px (small phone)
- Headings: Scale proportionally

**Preferred method:** Use relative sizing (em, rem) instead of fixed px

## Typography System Checklist

Before finalizing typography:

- [ ] Font sizes limited to 3-4 maximum
- [ ] Weight matches font's actual weights (not forced/generated)
- [ ] Line height matches content type (150% body, 100-120% titles)
- [ ] Body copy width between 45-75 characters
- [ ] Color contrast tested (minimum 4.5, preferably 7.0)
- [ ] Truncation strategy defined for long content
- [ ] Letter spacing adjusted if using geometric fonts (-2% to -4%)
- [ ] Mobile sizing considered (responsive scaling)
- [ ] Hierarchy clear (most important element obvious)
- [ ] All fonts included actual weights (not browser-generated)
