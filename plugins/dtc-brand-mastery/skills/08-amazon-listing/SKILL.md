## Amazon A9 Ranking Formula (Brandon Young -- BDSS 2025)

**Ranking Score = Performance x Relevancy**

### Performance Factors
- Click-through rate (CTR)
- Conversion rate
- Total revenue generated on that keyword

### Relevancy Factors
- Match type (exact match = strongest signal)
- Keyword location hierarchy:
  1. **Beginning of title** (strongest)
  2. Middle/end of title
  3. Bullet points
  4. Backend search terms
  5. Description (weakest)

### Why This Matters
- If relevancy score is 0.25 and performance score is 1,000 -> ranking score = 250 (page 2-3)
- Fix relevancy to 1.0 -> ranking score jumps to 1,000 (top 10)
- **Relevancy is a MULTIPLIER** -- poor relevancy destroys even excellent performance

### Three-Step SOP (Brandon Young)
1. **Fix conversion rate first** (images, price, reviews, copy)
2. **Fix indexing** -- keywords at rank 35-55 means Amazon is uncertain about relevance; rank 101+ means NOT indexed at all
3. **Then make spending decisions** using triangulated data (impression rank + conversion rate + organic rank)

## SEO-Optimized Title (Melisa Vong, Brandon Young)

**Formula:** Brand Name + Highest-Volume Keyword + Secondary Keywords + Key Benefits

- Max 200 characters including spaces (varies by category)
- **Lead with brand name** (required for Brand Registry SEO)
- Place highest-volume keyword as close to beginning as possible
- Must read naturally while being keyword-rich
- Use Helium 10 Scribbles tool to track keyword usage across fields

**Title Density Concept (Bradley Sutton):**
- Title density = how many page 1 listings have your exact search phrase in their title
- Low title density (2-3 out of 20) = easy to rank; putting keyword at title beginning can land page 1 within hours during honeymoon period
- High title density = harder but not impossible
- Always check title density before entering a niche

**Platform Disagreement (Brand Name in Title):**
- Amazon instructors: brand name first in title always (Brand Registry requirement)
- Michelle (TikTok): brand names may hurt TikTok Shop discovery
- **Resolution:** Amazon = brand name in title always. TikTok = test both. Shopify = brand in domain/header, benefit-driven product titles.

## Bullet Points (Melisa Vong)

- Use all 5 bullet points
- Total under 1,000 characters across all 5 bullets for readability
- **Format:** MAIN BENEFIT IN CAPS followed by descriptive explanation
- Each bullet tells a mini-story -- help customers visualize the benefit
- Sprinkle keywords throughout naturally
- Hit pain points AND multi-use applications (expands keyword targeting)
- Sell through emotion OR logic depending on customer persona

**Example:** Anti-snoring mouthguard bullets also target teeth grinding, bruxism, TMJ keywords -- expanding the addressable audience.

## Backend Search Terms (Melisa Vong)

- **250 byte limit** (not character limit -- special characters use more bytes)
- Use Helium 10 Frankenstein tool to remove duplicate words and compress
- Include keywords that don't fit naturally in visible listing
- Do NOT repeat keywords already in title/bullets (wastes bytes)
- Include misspellings, alternate names, related terms
- **Be cautious with irrelevant keywords** -- dilutes semantic relevance

## Semantic Relevance (Leo Sgovio -- BDSS 8)

**Core principle:** Including trendy but irrelevant keywords in your listing DILUTES semantic relevance and hurts rankings on your main keywords.

### Case Study
Fat burner ranked #1 for "Kim Kardashian weight loss pills" (22,000 reviews) but nowhere for "weight loss pills." A competitor with only 7,100 reviews was doing $250K/month vs. $90K/month because of stronger semantic relevance to core keywords.

### ChatGPT Semantic Relevance Audit
1. Take keyword list from Helium 10 tools
2. Prompt: "You are an NLP expert. Analyze semantic relevance of this keyword group."
3. Prompt: "Find the central topic. Build a final list excluding irrelevant and branded keywords, staying close to central topic."
4. Refine: "My product is [specific product]"
5. Final: "Exclude irrelevant/branded keywords, exclude no-commercial-intent keywords, exclude generic/informational keywords, order by semantic relevance, remove non-English keywords"

### Amazon Patent Insight
During cold start (new listing), Amazon relies heavily on listing text AND image content to determine relevance -- no behavioral data exists yet. Getting semantic relevance right from day one is critical.

## Listing Simplicity & Cognitive Load (Matt Kostan -- BDSS 2025)

- Stanford study: more choices = less willingness to buy
- Amazon study (2022): 28% of purchases done in 3 minutes; 50%+ done in 15 minutes
- Costco carries <4,000 products vs Walmart 150,000 = easier decisions = more sales

### Application to Listings
- One topic per image, dead simple messaging
- Anker's strategy: focus on portability + speed only (not 20 features)
- Face painting kit: simplified images = 10% to 13.5% conversion rate (nearly 40% increase)
- BBQ basket: addressed #1 concern ("does food fall out?") = 35% sales increase

### Product Highlights
Amazon rolling out "Product Highlights" -- AI-generated callouts on secondary images. Make sure your images communicate your benefits clearly since AI will extract them.

## Image Optimization for Rankings (Leo Sgovio -- BDSS 8)

**Amazon scans images for text content and uses it for ranking signals (documented in Amazon patents).**

### How to Optimize Images for SEO
1. Add target keyword phrases INTO image text (e.g., "not a magic weight loss pill" instead of "not a magic pill")
2. In Photoshop, add metadata: document title = brand name, description with star rating, keywords field, copyright notice
3. Export with "copyright and contact info" option selected
4. Save image filename with target keyword (e.g., "organic-green-tea-matcha-powder.jpg")
5. Applies to main images, secondary images, AND A+ content images
6. Google also indexes Amazon images using this metadata

### Image Best Practices
- Use all 7-9 image slots available
- Include infographic images showing benefits/features
- Include lifestyle images showing product in use
- Use Helium 10 Listing Analyzer "Media Comparison" to analyze competitor image strategies
- Identify niche theme (what's every competitor's 3rd image? 4th image?)

## Rufus & Cosmo Optimization (Vanessa Hung -- BDSS 2025)

- **Rufus:** Front-end AI shopping assistant that customers interact with
- **Cosmo:** Backend algorithm complementing A9; understands product function AND customer intent

### Optimization Process
1. Check Rufus-prompted questions on your listing AND competitors (may be 40-50 questions)
2. Collect all questions
3. Feed listing content + question list into ChatGPT
4. Prompt: "Does my listing answer these questions? Build scorecard 1-5"
5. Fix all scores below 3; add context about product functionality
6. Repeat this audit regularly -- Rufus questions evolve over time

### Image Intelligence
Images now feed Cosmo: an image showing 3 people in different sleeping positions teaches the system "all sleeping positions" without text. Visual content informs AI understanding of your product.

## Variation Strategy (Tomer Rabinovich -- BDSS 2023)

### The Problem with Tools
All Amazon tools (Helium 10 X-Ray, Jungle Scout, Review Insights) report WRONG variation-level sales. They report same BSR/sales across all variations multiplied by price -- meaningless for decision-making.

### Review Days Ratio -- Correct Method
1. Get reviews per variation from Keepa (under "Variations" tab -- use Ratings column, not Reviews)
2. Calculate reviews/day = reviews divided by days live
3. Multiply by total monthly sales to get correct per-variation sales

### Use Cases
- Which variation to launch first
- Which variations to add next
- How much inventory to order per variation
- Case study: product went from $25K/month to $125K/month just by launching the right variations

## Listing Troubleshooting Hacks (Kevin King)

| Problem | Solution |
|---|---|
| Can't change listing (blocked by another seller) | Open ticket asking to "remove the augmenter" (matches Seller Support SOPs) |
| Augmenter removal fails | Ask for "SKU Remover" -- removes ALL contributions (only if sole seller) |
| Product live but not in search | Ensure BOTH category AND item type fields completed in Vital Info (upload via flat file) |
| Vague listing issues | Ask support to run "TICO tool" on listing (internal diagnostic) |
| Need advanced support | Call Seller Support on Saturday to reach Costa Rica team (more capable than India team) |

### Additional Hacks
- **Spanish keywords:** US has second-largest Spanish-speaking population. Load top 20 ASINs into Cerebro, download keywords, use Google Sheets detect language + translate formulas, sort by "ES" for low-competition Spanish keywords
- **Forbidden keywords workaround:** Restricted words (CBD, etc.) can be indexed through Q&A section answers
- **A+ Premium Content hack:** Need 15 approved A+ content projects to unlock A+ Premium (free once unlocked)

## Anti-Patterns
- Stuffing irrelevant trending keywords into backend (destroys semantic relevance)
- Ignoring image SEO (Amazon patents show images affect ranking)
- Writing bullet points focused on features instead of benefits
- Repeating title keywords in backend (wastes 250-byte limit)
- Launching PPC before fixing listing conversion rate (wasted spend)
- Using tool-reported per-variation sales data (always wrong)
- Not checking Rufus questions regularly (AI shopping assistant evolves)
- Skipping title density analysis before entering a niche
