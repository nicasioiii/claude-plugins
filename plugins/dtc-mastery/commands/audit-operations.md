# Operations Efficiency Audit

## Step 1: Gather Operations Context

Ask the user these questions before proceeding. Do not skip any.

### Required Inputs
1. **Monthly revenue**: Approximate current monthly revenue?
2. **Team composition**: How many people work on the business? List roles (founder, VAs, full-time, agencies).
3. **Hours per week**: How many hours/week do YOU personally spend on the business?
4. **Top 5 time drains**: What are your 5 most time-consuming weekly tasks?
5. **SOPs**: Do you have documented processes? (none / some in docs / comprehensive wiki)
6. **Project management tool**: What do you use? (nothing, spreadsheets, Asana, Monday, ClickUp, other)
7. **Communication**: How does your team communicate? (email, Slack, WhatsApp, ClickUp, meetings)
8. **Technology stack**: List your main business tools (Shopify, Helium 10, Klaviyo, etc.)
9. **Biggest operational pain point**: What breaks most often or causes the most frustration?
10. **Customer support**: How do you handle customer inquiries? (personal email, Zendesk, Gorgias, other)

---

## Step 2: Score Each Category (Load: 14-operations-scaling)

### Category 1: Systemization Level (0-10)
Evaluate against Escala 5-level hierarchy:

| Score | Level | Description |
|---|---|---|
| 0-2 | No systems | Everything in founder's head, no documentation |
| 3-4 | Basic | Some processes written down, inconsistent format |
| 5-6 | Structured | L1-L3 mapped, some SOPs exist, project management tool in use |
| 7-8 | Systematic | Full process hierarchy, wiki + workflows, sprint methodology |
| 9-10 | Optimized | Escala-level systemization, 90-day reviews, continuous improvement |

### Category 2: Team Efficiency (0-10)
- Are tasks assigned to the right people (Four Zones alignment)?
- Is founder doing Zone of Incompetence or Competence tasks?
- Is team size appropriate for revenue level?
- Are there clear accountability assignments?
- Can the business operate without the founder for 2+ weeks?

### Category 3: Technology Stack (0-10)
- Is there a single source of truth for task management?
- Are tools consolidated or sprawling?
- Are there manual processes that should be automated?
- Is Excel/Google Sheets being used where bespoke tools would be better?
- Are there redundant tools doing the same thing?

### Category 4: Communication Efficiency (0-10)
- Is there a central hub (not email/Slack as primary)?
- Are meetings minimized in favor of async communication?
- Is information findable without asking someone?
- Are new team members able to self-serve answers?

### Category 5: Demand Forecasting & Inventory (0-10)
- Is sales velocity tracked (excluding stockout days)?
- Is seasonality accounted for in ordering?
- Are reorder points set with lead time buffer?
- Is there a 30/60/90 day forecast?

### Category 6: Customer Support Operations (0-10)
- Is support multi-channel (email, chat, phone, social)?
- Is first response time tracked?
- Are common questions handled with templates/automation?
- Is support data used to improve products/listings?
- Is there a review collection process post-support resolution?

### Overall Score: ___/60

| Score Range | Level | Priority |
|---|---|---|
| 50-60 | Optimized | Fine-tune, explore AI agents and advanced automation |
| 35-49 | Good | Fill specific gaps, implement quarterly reviews |
| 20-34 | Needs work | Prioritize systemization before scaling further |
| Below 20 | Critical | Pause growth, focus entirely on building systems |

## Step 3: Four Zones Task Audit (Load: 14-operations-scaling)

Take the user's top 5 time-consuming tasks and categorize each:

| Task | Zone | Recommended Action | Priority |
|---|---|---|---|
| [Task 1] | Incompetence / Competence / Excellence / Genius | Delegate / Automate / Keep | High / Med / Low |
| [Task 2] | ... | ... | ... |
| [Task 3] | ... | ... | ... |
| [Task 4] | ... | ... | ... |
| [Task 5] | ... | ... | ... |

Calculate: What % of founder's time is spent in Zone of Genius?
- Target: 60%+ of time in Zone of Genius
- If below 30%: critical delegation needed

## Step 4: Technology Audit (Load: 14-operations-scaling)

For each tool the user listed:
1. Map to business function (marketing, operations, finance, support, etc.)
2. Identify overlaps (multiple tools doing same thing)
3. Identify gaps (functions not covered by any tool)
4. Apply Start/Stop/Continue framework:
   - **Start**: tools needed but not yet adopted
   - **Stop**: tools that are redundant, underused, or creating confusion
   - **Continue**: tools that are working well

### Common Recommendations
- No project management tool? -> ClickUp (Wiki + Workflows)
- Using email for team communication? -> Migrate to ClickUp/Slack with structured channels
- Multiple spreadsheets for tracking? -> Evaluate bespoke alternatives
- No customer support platform? -> Gorgias (if Shopify) or Zendesk
- No shipping communication? -> Wonderment for proactive updates

---

## Step 5: Generate Improvement Roadmap

### Quick Wins (Week 1-2)
- Actions that can be completed in under 2 hours each
- High impact, low effort improvements
- Examples: set up project management board, create first SOP for most-repeated task

### Short-Term Improvements (Month 1)
- Systemize top 3 most time-consuming processes
- Consolidate redundant tools
- Set up basic demand forecasting
- Delegate first Zone of Incompetence task

### Medium-Term Improvements (Months 2-3)
- Build ClickUp Wiki structure (L1-L3)
- Sprint-document top processes
- Implement technology audit findings
- Hire for highest-priority delegation needs

### Long-Term Improvements (Months 3-6)
- Complete Escala 5-level documentation
- Implement 90-day quarterly review process
- Evaluate AI agent opportunities
- Optimize onboarding workflow

---

## Step 6: Output Report

Generate a complete operations audit report with:
1. Overall score with category breakdown
2. Four Zones analysis (% time in each zone)
3. Technology Start/Stop/Continue recommendations
4. Top 3 highest-impact improvements (prioritized)
5. 90-day improvement roadmap
6. Estimated time savings from recommended changes
7. Key metrics to track improvement (founder hours/week, response times, stockout frequency)
