# /build-offer-stack -- Complete Offer String Design

## INTAKE QUESTIONS (Ask All Before Building)

### Question 1: Main Product
**What is your main product?**
- Product name and description
- Current or intended price
- Delivery method (digital, physical, service, SaaS)

### Question 2: Customer Context
**Who buys this and what problem does it solve?**
- Target customer description
- Primary problem being solved
- What problems do customers hit NEXT after buying? (If unsure, we'll brainstorm)

### Question 3: Current State
**What do you have now?**
- A) Just the main product, nothing else
- B) Main product + order bump
- C) Main product + bump + 1 upsell
- D) Full stack but underperforming
- E) Starting from scratch

### Question 4: Business Model
**What type of business?**
- A) Digital info products / courses
- B) Physical products / DTC / supplements
- C) Service business / coaching
- D) SaaS / software
- E) Hybrid

### Question 5: Revenue Goal
**What monthly revenue are you targeting?** (Helps calibrate stack complexity)

## BUILD PROCESS

### Step 1: Front-End Offer Design (Load offer-stack-design)
- Select attraction offer type (or direct sale)
- Apply pricing psychology principles
- Validate front-end pricing against AOV targets

### Step 2: Order Bump Design (Load upsell-downsell-strategy)
- Design bump following the "accelerate the main product" rule
- Apply 4-part bump copy structure
- Set $27-$37 price range, 40%+ take rate target

### Step 3: Upsell Sequence Design (Load upsell-downsell-strategy)
- Map the "next problem" chain for each upsell
- Apply pricing architecture ($67-$97 -> $197 -> $44-$77)
- Select upsell types from Georgi's categories and Hormozi's 4 types
- Design downsell flow for each "no"

### Step 4: Backend Planning (Load backend-sequences)
- Sequence backend offers (under $100 first, then ascending)
- Plan 4-email sequence structure per product
- Set conversion rate targets by price point

### Step 5: Continuity Design (if applicable, load upsell-downsell-strategy)
- Select continuity type (Bonus, Discount, or Waived Fee)
- Apply Big Head Long Tail model if appropriate
- Plan downsell-the-upsell framework

### Step 6: Economics Validation (Load funnel-economics)
- Model complete stack economics
- Calculate expected AOV with take rate assumptions
- Validate LTGP:CAC ratio
- Check CFA level

### Step 7: Output
Deliver a complete offer stack:
1. **Offer Stack Map** -- Visual sequence of all offers
2. **Each Offer** -- Name, description, price, positioning, take rate target
3. **Downsell Flow** -- What happens at each "no"
4. **Backend Sequence** -- Offer order and timing
5. **Economics Model** -- AOV projection, CPA target, profitability estimate
6. **Build Order** -- Which offers to create first (sequential layering)
