# RSA Headline Creation & Iteration Loop

## THE 7 HEADLINE TYPES (30-CHARACTER MAX)

### Type 1: Relevance Anchor
Signal match to the searcher's query. Use Dynamic Keyword Insertion.
- Formula: {KeyWord:Fallback Text}
- Example: "{KeyWord:Home Security Systems}"
- Purpose: Ensures ad headline mirrors what the user searched
- Note: The fallback text shows when the keyword is too long

### Type 2: Problem / Pain Recognition
Mirror the searcher's frustration. Most powerful type for emotional connection.
- Formula: [Tired of / Stop / End / No More] + [Specific Problem]
- Examples: "Stop Losing Sleep Over Bills" / "End Chronic Back Pain Now" / "No More Wasted Ad Spend"
- Purpose: Shows you understand their world

### Type 3: Value Proposition
What you offer + the primary benefit. Clear in under 5 seconds.
- Formula: [Specific Offer] + [Primary Benefit]
- Examples: "Custom Meal Plans in 5 Min" / "AI Bookkeeping, Zero Errors"
- Purpose: Communicates the core promise

### Type 4: CTA (Call to Action)
The exact next step. Action verb + specific outcome.
- Formula: [Action Verb] + [Specific Outcome] + [Optional Modifier]
- Examples: "Get Your Free Audit Today" / "Start Saving in Minutes" / "Book Your Demo Now"
- Purpose: Drives the click

### Type 5: USP / Benefit
Differentiate from competitors with a quantified benefit.
- Formula: [Differentiator] + [Quantified Benefit]
- Examples: "30% Faster Than [Category]" / "Save 12 Hours Per Week" / "Made in USA, Ships Free"
- Purpose: Answers "why you and not them?"

### Type 6: Social Proof & Authority
Trust through evidence. Use specific numbers, not vague claims.
- Formula: [Specific Metric] + [Audience/Context]
- Examples: "Trusted by 50,000+ Teams" / "4.9 Stars, 2,800 Reviews" / "Used by Fortune 500"
- Purpose: Reduces perceived risk

### Type 7: Risk Removal
Lower the barrier to action. Specific guarantee terms.
- Formula: [Guarantee Type] + [Reassurance Statement]
- Examples: "Full Refund, No Questions" / "Free 30-Day Trial" / "Cancel Anytime, Zero Risk"
- Purpose: Eliminates the last objection

---

## RSA STRUCTURE RULES

- **6-8 headlines per RSA** -- NOT 15 (data poverty problem)
- All 7 types represented across the slots
- H1 = Relevance Anchor (keyword slot, changes per ad group)
- H2-H8 = Mix of remaining 6 types (identical across ad group cluster)
- 25-30 characters per headline (maximize the space)
- 2-3 descriptions per RSA (80-90 chars each)
- **One RSA per ad group** -- no competing RSAs splitting traffic

### Data Poverty Problem
- 15-headline RSA = up to 2,730 possible three-headline combinations
- Most ad groups never accumulate enough impressions for Google to learn which combinations work
- Solution: Limit to 6-8 headlines, template across clusters, aggregate data

## THE ITERATION LOOP (4-PILLAR SYSTEM)

### Pillar 1: Templatize
1. Group ad groups into clusters with similar intent (e.g., all "home security" variations)
2. Create RSA template: assign each headline slot a specific type
3. H1 = keyword-specific (DKI), all other slots identical across the cluster
4. This creates: 20 ad groups x 5,000 impressions = 100,000 aggregated impressions for one angle
5. Result: Enough data to evaluate headline performance

### Pillar 2: Aggregate
Pull asset-level performance across the entire cluster, NOT per ad group.

**Metrics to calculate:**
- CPI (Conversions Per Impression) -- primary metric
- RPI (Revenue Per Impression) -- for e-commerce
- PPI (Profit Per Impression) -- if margin data available
- AIS (Ad Impression Share) -- visibility metric

**Do NOT use:**
- CTR as primary metric (high CTR can attract unqualified clickers)
- Google's "Low / Good / Best" labels (they are CTR-driven, not outcome-driven)

### Pillar 3: Diagnose (Performance Quadrants)

| | High Impression Share | Low Impression Share |
|---|---|---|
| **High CPI/RPI** | **Winners** -- Scale these. They work AND Google shows them. | **Hidden Gems** -- Pin these to positions. Google isn't showing them enough. |
| **Low CPI/RPI** | **Silent Killers** -- Remove FIRST. Google shows them a lot but they don't convert. Actively hurting your account. | **Underperformers** -- Replace. Low visibility + low conversion = no value. |

**Priority order:** Remove Silent Killers > Pin Hidden Gems > Scale Winners > Replace Underperformers

### Pillar 4: Iterate
Testing hierarchy -- start broad, then narrow:

**Phase 1 (Foundation):** Test headline TYPES against each other
- Which of the 7 types produces best CPI for your account?
- Run for 2-4 weeks with sufficient volume

**Phase 2 (Growth):** Test sub-angles within winning types
- If "Problem/Pain" type wins, test different pain points
- If "Social Proof" type wins, test different proof formats

**Phase 3 (Mastery):** Test specific asset variations within winning sub-angles
- Word-level optimization of proven angles
- A/B variations of your best-performing headlines

## DESCRIPTION WRITING

### Structure
- 2-3 descriptions per RSA
- 80-90 characters each (use the space)
- Descriptions expand on headline promises with supporting detail

### Description Types
1. **Benefits-focused:** Expand on the value proposition with specifics
2. **Trust-focused:** Social proof, guarantees, credentials in longer format
3. **Action-focused:** Elaborate CTA with urgency or specifics

### Example Description Set
1. "Join 50,000+ businesses saving 12+ hours per week on reporting. Start free today."
2. "No credit card required. Full access for 30 days. Cancel anytime with zero fees."
3. "Automate your monthly reports in minutes, not hours. See the dashboard in action."

---

## CREATIVE EXCELLENCE HIERARCHY

When diagnosing ad performance, check in this order:

1. **Offer** -- Is the offer compelling? (Irresistible offer > mediocre offer with great copy)
2. **Message** -- Does the message match the audience's awareness level?
3. **Awareness Level Match** -- Are you using the right type of messaging for where the audience is?
4. **Assets** -- Are the headlines, descriptions, and images optimized?

Do NOT optimize assets (level 4) while the offer (level 1) is broken.

## TESTING METHODOLOGY (Expanded)

### Phase 1: Type Testing (Weeks 1-4)
**Goal:** Determine which of the 7 headline types produces the best CPI for your account.
- Create one RSA template with all 7 types represented
- Deploy across 10+ ad groups in a cluster
- Aggregate CPI data at the type level (not ad group level)
- After 4 weeks with sufficient volume: identify the top 2-3 performing types

### Phase 2: Angle Testing (Weeks 5-8)
**Goal:** Test different angles within your winning types.
- Keep your winning type in H1 or H2 position
- Create 3-4 variations of the winning type with different angles
- Example: If "Problem/Pain" type won, test:
  - Pain about time ("Stop Wasting Hours on...")
  - Pain about money ("Stop Losing Money to...")
  - Pain about quality ("Stop Settling for Bad...")
  - Pain about complexity ("Stop Struggling With...")
- Aggregate again; identify the winning angle

### Phase 3: Asset Optimization (Weeks 9+)
**Goal:** Optimize the specific wording of your best-performing headlines.
- Take your winning type + winning angle combination
- Create 2-3 word-level variations
- Example: "Stop Wasting Hours on Reports" vs "Cut Report Time by 80%" vs "Reports Done in Minutes"
- This is where marginal gains happen -- small wording changes can move CPI significantly

### Testing Rules
- Never test more than one variable at a time (type OR angle OR wording, not all three)
- Minimum 1,000 impressions per headline before evaluating
- Use CPI (Conversions Per Impression) as primary metric -- NOT CTR
- Google's "Low / Good / Best" labels are CTR-driven and misleading for conversion optimization
- Remove Silent Killers (high impression share + low CPI) before doing anything else

## PINNING STRATEGY

### When to Pin Headlines
- Pin H1 to Position 1 when you need keyword relevance guaranteed in every impression
- Pin a Hidden Gem to Position 2 or 3 to force Google to show it more often
- Never pin more than 2 positions (leaves enough flexibility for Google to optimize)

### Pinning Decision Framework
| Headline Status | Action |
|---|---|
| Winner (high CPI, high impression share) | Do NOT pin -- Google is already showing it |
| Hidden Gem (high CPI, low impression share) | PIN to Position 2 or 3 |
| Silent Killer (low CPI, high impression share) | REMOVE immediately |
| Underperformer (low CPI, low impression share) | REPLACE with new variation |
