# Meta Simple Method - Complete SOP

## Overview
- **Purpose:** Exit Facebook Learning Phase with single high-budget campaign using catalog + open targeting
- **Timeline:** 7 days to profitability
- **Minimum Account Age:** 2+ years with trained pixel
- **Best For:** E-commerce with 500+ product catalogs
- **Expected ROAS:** 2.0-3.0+

---

## PREREQUISITES CHECKLIST

Before launching Simple Method, verify:

- [ ] Account is 2+ years old (Facebook has account history/quality score)
- [ ] Pixel has been collecting purchase data for 12+ months
- [ ] Product catalog is 500+ products (can validate up to 1,000+)
- [ ] Catalog has NO technical bugs (products showing, prices correct, images loading)
- [ ] You've analyzed buyer data (know age/gender mix of actual customers)
- [ ] Profit margin is 50%+ (otherwise scaling won't work)
- [ ] ROAS target calculated (based on profit margin with 30% buffer)

---

## BUDGET CALCULATION

**Step 1: Determine Target CPA**
```
Profit Margin Example: 62.5%
Break-Even ROAS: 1.6
Target ROAS (with 30% buffer): 1.6 × 1.3 = 2.1
Target ROAS goal: 2.5+ (safe zone)

AOV: $80
Desired Profit Margin: $30 (37.5% margin)
Target CPA: AOV ÷ Target ROAS = $80 ÷ 2.5 = $32 CPA
```

**Step 2: Calculate Minimum Daily Budget**
```
Formula: (Target CPA × Desired Sales) ÷ Days

Example:
- Goal: 50 sales in 7 days = 7 sales/day
- Target CPA: $32
- Minimum Daily Budget: ($32 × 7) ÷ 7 = $224/day

Or:
- Goal: 70 sales in 7 days = 10 sales/day
- Minimum Daily Budget: ($32 × 10) ÷ 7 = $457/day
```

**Rule of Thumb:**
- Conservative: $100-150/day minimum
- Recommended: $200-400/day (ensures enough spend to hit 50 conversions)
- Aggressive: $500+/day (fast Learning Phase exit but higher risk)

## CAMPAIGN SETUP (STEP-BY-STEP)

### Campaign Level
1. **Campaign Name:** "[Simple Method] - [Date]" (Example: "[Simple Method] - 03-06-2026")
2. **Campaign Objective:** Conversions (Catalog Sales)
3. **Campaign Budget Optimization (CBO):** Enabled (budget set at campaign, not ad set level)
4. **Daily Budget:** Use calculated amount from above (Example: $300/day)
5. **Bid Strategy:** Automatic (Facebook optimizes bids)
6. **Start Date:** Immediate
7. **End Date:** None (let it run; you'll scale or pause manually)

### Ad Set Level
1. **Ad Set Name:** "Open Targeting - All Age/Gender"
2. **Audience:**
   - **Audience Type:** Everyone / Saved Audience (select "Broad")
   - **Locations:** United States (or your primary market)
   - **Age:** Based on actual buyer analysis (Example: 30-65+)
   - **Gender:** Based on buyer data (Example: Women if 70% buyers are female)
   - **Detailed Targeting:** NONE (leave blank for true open targeting)
   - **Exclusions:** None (don't exclude past customers; retarget separately)
3. **Placements:** All placements enabled (Feeds, Reels, Stories, Messenger, Audience Network)
4. **Optimization Event:** Conversions (Purchase)
5. **Attribution Window:** 7-day click (or 1-day if in volatile testing phase)

### Creative Assets
1. **Campaign Type:** Catalog (Dynamic Product Ads - DPA)
2. **Catalog Source:** Your Merchant Center product catalog
3. **Product Selection:** Dynamic (Facebook auto-selects products to test)
4. **Template:** Standard carousel or single image
5. **Primary Image:** Use best-performing image from previous campaigns
6. **Headline:** Keep simple; focus on offer or value prop
7. **Ad Copy:** Problem-Agitation-Solution-Benefit format (80 characters max for above-fold visibility)

## SCALING WORKFLOW (DAY 1-7+)

### Days 1-3: Initial Data Collection
- **Check frequency:** 1-2 times daily
- **Monitor:** Impressions (should see 10K+), Clicks, Cost Per Click
- **Expected state:** Unstable metrics; don't react yet
- **Action:** Do nothing. Let algorithm collect initial data.
- **Red flag:** Zero sales by Day 2 morning = possible pixel issue; check implementation

### Days 4-7: Learning Phase Continuation
- **Check frequency:** 1 time daily (morning)
- **Monitor:** Total conversions (target: 50+ by Day 7)
- **Expected metrics:**
  - CPA will be volatile (±30% daily variance normal)
  - ROAS will fluctuate (2.0-3.5 range common)
- **Action:** Pause if CPA is consistently 2x+ your target for 2 consecutive days
- **Learning Phase exit criteria:** 50+ conversions AND 7 days elapsed AND ROAS > target

### Day 8-14: Post-Learning Phase Scaling
**Once Learning Phase is exited:**

**Daily Scaling Formula:**
```
Day 8: $300 × 1.10 = $330
Day 9: $330 × 1.10 = $363
Day 10: $363 × 1.10 = $399
Day 11: $399 × 1.10 = $439
Day 12: $439 × 1.10 = $483
Day 13: $483 × 1.10 = $531
Day 14: $531 × 1.10 = $584
```

**When to STOP scaling:**
- CPA increases 20%+ from post-Learning Phase baseline
- ROAS drops below 1.5 (loss territory)
- Reach plateau (same # impressions, declining conversions)

**When to KILL the campaign:**
- ROAS drops below 1.2 for 2+ consecutive days
- Spending full budget with negative ROAS (losing money)
- Pixel issue detected (no data being recorded)

### Day 15+: Mature Campaign Management

**Scaling Rules (Post-Week 2):**
- Continue 10% daily scaling IF:
  - ROAS remains 2.0+
  - CPA stable or declining
  - Reach continuing to grow

- Switch to 7-10% scaling IF:
  - Saturation signs appear (CPA rising)
  - ROAS declining (2.0 → 1.8 → 1.6 trend)

- Pause scaling IF:
  - ROAS drops below target ROAS
  - CPA exceeds 150% of target
  - Reach reaches plateau (same reach, declining sales)

**Duplication Strategy:**
- Once campaign hits $300-400/day with ROAS 2.0+, duplicate it
- Name duplicate: "[Simple Method] - Run 2"
- Run both simultaneously (don't combine into larger single campaign)
- Each duplicate gets independent Learning Phase
- Expected: 2 campaigns × $300/day = $600/day total spend with better stability
- Can duplicate up to 3-4 times if performance maintains

## DAILY MONITORING CHECKLIST

**Every morning (takes 5 minutes):**

- [ ] Check total campaign ROAS (look for 1.5+ minimum)
- [ ] Check CPA (compare to target; acceptable if within 20% variance)
- [ ] Check daily budget spent (should be close to $X/day you set)
- [ ] Check conversion count (on track for continued sales?)
- [ ] Check if new scaling day is needed
  - IF ROAS > target AND CPA stable → Increase budget 10%
  - IF ROAS declining trend → Don't scale; pause for 2 days
  - IF ROAS good but spending less than budget → No action needed

**Every Friday (weekly review):**

- [ ] Calculate rolling 7-day ROAS
- [ ] Identify if saturation is beginning (CPA trend up?)
- [ ] Review top-performing products (are some driving all sales?)
- [ ] Decide: Continue scaling, hold budget, or pause?
- [ ] Plan next week's budget ceiling

## TROUBLESHOOTING

### Problem: Zero sales on Day 2
**Diagnosis:** Pixel implementation issue, poor audience targeting, or weak creative
**Solution:**
1. Check pixel is firing (use Pixel Helper browser extension)
2. Verify campaign is targeting right demographic (check Audience size)
3. Check creative: Is headline visible? Does copy make sense?
4. Give it to Day 3 morning before killing

### Problem: ROAS 1.0-1.2 (breakeven or losing)
**Diagnosis:** Either audience is wrong, margins are wrong, or account quality is poor
**Solution:**
1. Verify profit margin calculation (are numbers actually $30+ per unit?)
2. Check if account quality is the issue (new account = poor traffic quality)
3. Try tighter targeting: Instead of 30-65+, test 35-55 (core buyers)
4. Give it 3 more days; if no improvement, kill and audit account quality

### Problem: High ROAS (2.5+) but spending slow (only $50/day of $300 budget)
**Diagnosis:** Audience is small, or algorithm is being cautious
**Solution:**
1. Expand age range (30-65+ instead of 40-55)
2. Remove gender restriction if possible (test all genders)
3. Wait 2-3 days; algorithm may just need time to find more people
4. Increase budget incrementally ($300 → $350) instead of 10% formula

### Problem: ROAS was 2.5, now 1.4 (collapsed)
**Diagnosis:** Audience saturation, algorithm regression, or external factor (iOS privacy)
**Solution:**
1. First check: Did you make any targeting changes? Revert if yes.
2. Check: Is it a bad period? Low-traffic weekday? This may be temporary.
3. Pause the campaign for 2 days; often re-training helps
4. If it stays low after 2-day pause, kill the campaign
5. Start new campaign with different audience or targeting approach

### Problem: Pixel shows "Insufficient Data"
**Diagnosis:** Pixel hasn't received 50+ conversions to build lookalike audiences
**Solution:** This is normal for new campaigns. Simple Method doesn't rely on lookalikes (uses open targeting), so continue scaling. Data sufficiency will come after 100+ conversions.

## SUCCESS INDICATORS

**Campaign is working well if:**
- ✓ ROAS 2.0+ achieved by Day 7
- ✓ CPA within 20% of target
- ✓ Consistent daily sales (not 0-5 one day, 20 next)
- ✓ Reach growing daily (more people seeing ads)
- ✓ Can scale 10% daily for 2+ weeks without ROAS collapse

**Campaign should be killed if:**
- ✗ ROAS below 1.5 by Day 10
- ✗ CPA 2x+ target even after full Learning Phase
- ✗ Pixel not recording purchases (data shows 0 conversions but you know sales happened)
- ✗ ROAS was good, now declining 3 days straight (saturation)

## EXPECTED CAMPAIGN LIFESPAN

**Typical Simple Method Campaign Timeline:**

| Phase | Duration | Actions |
|-------|----------|---------|
| **Learning Phase** | Days 1-7 | Collect 50+ conversions, accept volatility, don't scale |
| **Stable Growth** | Days 8-21 | 10% daily scaling, monitor for saturation |
| **Maturity** | Days 22-60+ | Slower scaling (5-7%), watch for saturation |
| **Fatigue** | After 60+ days | ROAS declining; creative fatigue likely; prepare replacement |

**Duplication Timeline:**
- Week 2: Duplicate original campaign once
- Week 3: If both performing, duplicate again (now 3 campaigns)
- Week 4+: Max 3-4 simultaneously; any more = cannibalization

## VARIATIONS & TESTING

### Variation 1: Tighter Demographics
**Instead of:** Ages 30-65+, all genders
**Test:** Ages 40-60, women only

**Why:** If 80% of customers are 40-60 year old women, tighter targeting = lower CPC

**When to test:** After main campaign hits profitability; run as separate campaign

### Variation 2: Mobile-Only Placement
**Instead of:** All placements
**Test:** Mobile feeds + reels only (exclude desktop)

**Why:** Mobile placements often have lower CPC
**Trade-off:** Reach is smaller, but CPA may be lower

### Variation 3: Lookalike Audience (Post-Profitability)
**Once:** Main campaign has 500+ conversions
**Test:** 1% Lookalike (Purchase, 180-day window) instead of open targeting

**Why:** Lookalike captures warm audience; often scales better
**Expected:** ROAS may dip 5-10% initially, then stabilize higher

---

## KEY TAKEAWAYS

1. **One campaign, one audience, open targeting** = simplicity = easier to scale
2. **Always calculate budget from CPA math**, not guessing
3. **10% daily scaling works** because it avoids re-entry risk
4. **Duplication > Budget increases** = run 2-3 $300 campaigns instead of 1 $900 campaign
5. **Monitor daily, but don't overreact** to single-day volatility; look at 7-day trends
6. **Kill fast if metrics are wrong** by Day 10; don't let it bleed budget indefinitely
