# Content Architecture -- Complete Reference

## PILLAR-CLUSTER MODEL

### Core Concept
One comprehensive pillar page covers a broad topic. Multiple cluster posts explore subtopics in depth. Internal linking creates a coherent content ecosystem.

### Pillar Page Requirements
- Covers the broad topic comprehensively (3,000-5,000+ words)
- Links OUT to every cluster page in the topic group
- Receives links FROM every cluster page back to it
- Targets the highest-volume keyword in the topic group
- Updated regularly as new cluster content is added
- Placed prominently in site navigation

### Cluster Page Requirements
- Covers a specific subtopic in depth (1,500-3,000 words typical)
- Links UP to the pillar page with keyword-rich anchor text
- Links LATERALLY to related cluster pages when topically relevant
- Targets a specific long-tail or secondary keyword
- Can stand alone as valuable content

### Internal Link Flow
```
Pillar Page (broad keyword)
  |-- Cluster 1 (subtopic keyword) <-> Cluster 2
  |-- Cluster 3 (subtopic keyword) <-> Cluster 4
  |-- Cluster 5 (subtopic keyword)
```

## SOFT SILO MODEL (Authority Hacker)

### Architecture
- Content organized into WordPress categories (not rigid URL silos)
- Hub page per category serves as the pillar
- Breadcrumbs: Home > Category > Post
- Navigation menu reflects category structure
- Cross-silo linking permitted when topically relevant

### Why "Soft" Silo
Unlike hard silos (where content ONLY links within its silo), soft silos allow contextually justified cross-links. This matches how real topics interconnect and avoids artificially constraining link flow.

### Site Planning Process
1. Expand competitor list (search niche, note all ranking sites)
2. Research monetization (find all affiliate programs, commission rates)
3. Expand keyword list (brainstorm + Ahrefs + competitor mining)
4. Reverse engineer competitor keywords
5. Plan skyscraper content (linkable assets for link building)
6. Create sitemap (full site plan mapping content to categories)
7. Plan internal links between categories where topics overlap

## THREE-LAYER ARCHITECTURE (Koray)

### Root Layer
- Connected to homepage
- Links to ALL seed articles
- Receives ALL PageRank from homepage
- Highest quality articles on the site
- Uses central search intent in H1
- Example: "What to Know Before Going to Germany"

### Seed Layer
- Mid-level articles
- Link UP to root and DOWN to nodes
- Serve as category-level content
- Example: "Germany Visa Types"

### Node Layer
- Individual entity-attribute articles
- Link UP to seeds (rarely directly to root)
- Most numerous layer
- Example: "Germany Student Visa Requirements"

### Key Differences from Pillar-Cluster
Koray uses BOTH taxonomy (hierarchical parent-child) AND ontology (relational cross-links) simultaneously. This is more sophisticated than simple hub-and-spoke.

## CONTENT TYPES FOR SITE ARCHITECTURE

### Authority Hacker's Three Types
| Type | Purpose | Examples | Typical % |
|---|---|---|---|
| **Info content** | Traffic + link building | Question posts, list posts, how-to guides | ~50-60% |
| **Commercial content** | Revenue + conversions | Reviews, roundups, vs, alternatives | ~30-40% |
| **Support content** | Internal linking glue | Short explainers, glossary entries | ~10% |

### Vahe Arabian's Three Buckets
| Bucket | Purpose | Format | Execution |
|---|---|---|---|
| **Informational** | Drive traffic (TOFU) | How-to, guides, tutorials | Highest volume |
| **Promotional** | Earn links (connective tissue) | Listicles, curated content, data pieces | Link-worthy assets |
| **Tactical** | Drive conversions | Reviews, comparisons, product recommendations | Revenue-focused |

Execute all three in synergy. Run in quarterly campaign sprints.

## SITEMAP PLANNING (Authority Hacker)

### Pre-Build Planning
- Map EVERY planned content piece before building the site
- Plan internal linking structure in advance
- Identify which commercial pages each info page should link to
- Plan hub pages for each main category

### Master Content Database
After building, maintain a spreadsheet tracking:
- URL, title, keyword target, word count
- Category, content type (info/commercial/support)
- Publication date, last updated date
- Internal links TO this page (count)
- Internal links FROM this page (count)
- Rankings, traffic, backlinks (updated monthly)

## CATEGORY PLANNING

### WordPress Category Best Practices
- 5-8 main categories for most sites (not too many, not too few)
- Each category should contain minimum 5-10 articles to justify existence
- Category pages should have optimized titles, descriptions, and content
- Use parent/child categories for depth (max 2 levels recommended)
- Assign each post to ONE primary category (avoid multi-category assignment)

### E-Commerce Category Architecture (Cromwell)
- Start with collections, not products
- Collections rank easier than individual products
- Use sub-collections to multiply keyword coverage (same products, different keywords)
- Category deduplication: tag pages and filter pages should be noindexed

## QUARTERLY CAMPAIGN SPRINTS (Vahe Arabian)

### Sprint Structure
1. **Plan** (Week 1): Keyword research, competitor analysis, content brief creation
2. **Produce** (Weeks 2-8): Writing, editing, optimizing
3. **Publish** (Weeks 9-10): Batch publishing with internal links
4. **Promote** (Weeks 11-12): Outreach, social, syndication
5. **Measure** (Week 13): Performance review, adjust next sprint

### Velocity Progression
- Quarter 1: Establish baseline process, produce minimum viable content
- Quarter 2: Increase velocity 25-50% as systems mature
- Quarter 3: Further increase; start content repurposing
- Quarter 4: Audit year's content, plan next year based on performance data

## PUBLISHER SEO SURFACES (Vahe Arabian)

For media/publisher sites, optimize across multiple content surfaces:
- Google News SEO (structured data, article freshness)
- Google Discover (engaging headlines, high-quality images)
- Standard Search (evergreen + news content mix)
- Video surfaces (YouTube, Google Video carousel)
- AMP (Accelerated Mobile Pages -- declining importance but still used)

Each surface has different optimization requirements and content format preferences.
