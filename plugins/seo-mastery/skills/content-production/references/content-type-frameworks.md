# Content Type Frameworks

Master five content types that rank and convert: guides, reviews, comparisons, how-tos, and roundups.

## Framework 1: Comprehensive Guides (Pillar Content)

**Purpose**: Establish topical authority; rank for broad keywords; become reference page

**Best For**: Main topics (natural skincare, retirement planning, dog training)

**Typical Word Count**: 2,500-5,000 words

**Structure**:
```
H1: Ultimate Guide to [Topic]

Intro (200-300 words):
- Hook: Why this topic matters
- What reader will learn
- Table of contents (jump links)

H2: Foundational Knowledge (600-800 words)
- What is [topic]
- Why it matters
- Common misconceptions

H2: Core Concept #1 (800-1,000 words)
H3: Aspect A
H3: Aspect B
H3: How it applies to reader

H2: Core Concept #2 (800-1,000 words)
H3: Aspect A
H3: Aspect B

H2: Core Concept #3 (600-800 words)

H2: Common Mistakes (400 words)
- Mistake 1: Why it matters
- Mistake 2: Alternative approach
- Mistake 3: How to avoid

H2: Implementation Guide (500-600 words)
- Step 1: ...
- Step 2: ...
- Step 3: ...

H2: FAQ (300-400 words)
- Q1: A1
- Q2: A2
- Q3: A3

Conclusion (200-300 words):
- Recap main points
- Restate benefit
- Clear call-to-action
```

**SEO Strategy**: Comprehensive guides attract backlinks because they're reference-quality. Other sites link to them as sources.

**Monetization**: Link from guides to money pages (product pages, services, affiliate products)

## Framework 2: Product Reviews (Money Pages)

**Purpose**: Drive affiliate revenue; rank for commercial keywords

**Best For**: Products you've tested or researched

**Typical Word Count**: 1,500-2,500 words

**Structure**:
```
H1: [Product Name] Review | [Unique Angle]

Intro (150-200 words):
- Hook: Why you tested this
- Who it's for
- Bottom line verdict (upfront)

H2: Overview (150-200 words)
- What is [product]
- Key specs
- Price range
- My rating: 4.5/5 stars (or verdict)

H2: Pros (300-400 words)
- Pro 1: Explanation with example
- Pro 2: Explanation with example
- Pro 3: Explanation with example
- Pro 4: Explanation with example

H2: Cons (300-400 words)
- Con 1: How to work around
- Con 2: Who this affects
- Con 3: Minor complaint

H2: How It Compares (400-500 words)
H3: vs [Competitor 1]: [Key difference]
H3: vs [Competitor 2]: [Key difference]
H3: vs [Competitor 3]: [Key difference]

H2: Detailed Testing & Results (500-600 words)
- How I tested it (real experience)
- Specific results (numbers)
- Use cases it's best for
- Who should buy it

H2: Who Should Buy (200-300 words)
- Best for: [Audience 1]
- Good for: [Audience 2]
- Skip if: [Audience 3]

H2: FAQ (300 words)
- Common questions answered

H2: Final Verdict (150-200 words)
- Restate who it's for
- Bottom line
- Clear CTA ("Buy on Amazon" or link)
```

**Monetization**: Affiliate links to product (Amazon Associates, brand programs)

**Key Element**: Personal testing/experience (not just rewording product page)

## Framework 3: Comparison Articles (Money Pages)

**Purpose**: Rank for "vs" keywords; help readers choose; drive affiliate revenue

**Best For**: Comparing 2 popular products/methods

**Typical Word Count**: 2,000-3,000 words

**Structure**:
```
H1: [Option A] vs [Option B]: Complete Comparison

Intro (200 words):
- Setup: Why people compare these
- What you'll learn
- Quick verdict

H2: Quick Comparison Table (150 words)
- Side-by-side table with key specs
- Price, features, best for, rating

H2: What is [Option A]? (400-500 words)
- Explanation
- How it works
- Key benefits
- Best use cases
- Pricing

H2: What is [Option B]? (400-500 words)
- Explanation
- How it works
- Key benefits
- Best use cases
- Pricing

H2: Head-to-Head Comparison (800-1,000 words)
H3: Price: Which is cheaper? (pros/cons of each)
H3: Features: What's included? (comparison)
H3: Ease of Use: Which is simpler? (examples)
H3: Performance: Which performs better? (specs)
H3: Customer Support: Who has better support?
H3: Learning Curve: How long to master?

H2: The Verdict: Which Should You Choose? (300-400 words)
H3: Choose [Option A] if... (specific criteria)
H3: Choose [Option B] if... (specific criteria)
H3: Choose Both if... (scenarios)

H2: Detailed Use Cases (400-500 words)
- Beginner: Which is better
- Advanced: Which wins
- [Specific use case]: Recommendation

H2: FAQ (250 words)

Conclusion (150 words):
- Recap key differences
- Recommendation based on reader type
- CTA to products
```

**Best Practices**:
- Don't show bias (acknowledge both have merits)
- Use real data/testing
- Answer the actual question (don't dodge picking a winner)

## Framework 4: How-To / Tutorial (Supporting Content)

**Purpose**: Build topical authority; rank for "how to" keywords; earn backlinks

**Best For**: Processes, techniques, step-by-step instructions

**Typical Word Count**: 1,500-2,500 words

**Structure**:
```
H1: How to [Accomplish Goal] | [Benefit]

Intro (150-200 words):
- Hook: Why learn this
- What you'll accomplish
- How long it takes
- What you'll need preview

H2: What You'll Need (200-300 words)
- Tools required (specific brand/model)
- Materials
- Prep work
- Cost estimate

H2: Overview of the Process (150-200 words)
- Big picture: How it works
- Why this method is best
- Expected results

H2: Step-by-Step Instructions (1,000-1,200 words)
H3: Step 1: [First action]
- Detailed explanation
- Common mistakes
- Pro tip
- Image showing

H3: Step 2: [Second action]
- [Same structure]

H3: Step 3: [Third action]
H3: Step 4: [Fourth action]
[Continue as needed]

H2: Pro Tips for Success (300-400 words)
- Tip 1: [Shortcut or best practice]
- Tip 2: [How to avoid common mistake]
- Tip 3: [Advanced technique]

H2: Troubleshooting (300 words)
- Problem 1: Solution
- Problem 2: Solution
- Problem 3: Solution

H2: What's Next? (200-300 words)
- Advanced techniques
- Related topics
- How to apply this

H2: FAQ (250 words)

Conclusion (150 words):
- Encouragement
- Recap key points
- Next steps
```

**Key Element**: Real, detailed steps (not vague overview)

**Monetization**: Low-direct; high-indirect (links to money pages, builds authority)

## Framework 5: Roundup / Best Of (Money Pages)

**Purpose**: Rank for "best [X]" keywords; review multiple products; affiliate revenue

**Best For**: Comparing 5-10 similar products or options

**Typical Word Count**: 2,000-3,000 words

**Structure**:
```
H1: Best [Product Type] for [Use Case] | 2024 Guide

Intro (200-300 words):
- Hook: How to choose
- Selection criteria (what we looked for)
- Quick overview of top picks
- Buyer type summary

H2: Quick Comparison Table (150 words)
- Product | Price | Best For | Rating

H2: Our Selection Criteria (200-300 words)
- What we tested for
- How we evaluated
- Why these stand out

H2: Top Pick: [Product #1] (300-400 words)
- Overview and key specs
- Why we chose it
- Best for: [Who/what]
- Pros and cons
- Price point
- Where to buy (affiliate link)

H2: [Product #2] (250-350 words)
[Same structure as #1]

H2: [Product #3] (250-350 words)

H2: [Product #4] (250-350 words)

H2: [Product #5] (250-350 words)

H2: Comparison Table (100 words)
- Side-by-side detailed specs

H2: How to Choose (300-400 words)
- For beginners: Recommendation
- For advanced: Recommendation
- On a budget: Recommendation
- Premium option: Recommendation

H2: FAQ (250-300 words)

Conclusion (150-200 words):
- Recap selections
- Final recommendation
- CTA
```

**Key Element**: Real testing/experience with products (not just research)

**Monetization**: Affiliate links to all products compared

## Content Type Selection Matrix

| Content Type | Best For | Keyword Type | Ranking Difficulty | Affiliate Potential |
|---|---|---|---|---|
| Guide | Broad topics, topical authority | Informational | Medium | Low |
| Review | Single products | Commercial | High | High |
| Comparison | 2 similar options | Commercial | High | High |
| How-To | Processes, techniques | Informational | Medium | Low |
| Roundup | Multiple similar products | Commercial | Medium-High | Very High |

**Strategy**:
- 20% guides (authority building, backlink targets)
- 20% how-tos (authority building, backlink targets)
- 60% reviews/comparisons/roundups (revenue-focused)

## Template Customization by Niche

### E-Commerce Product Focus

**Most effective**: Reviews, Roundups, Comparisons
- Target "best [product] for [use case]"
- Affiliate links to Amazon/brand sites
- Test products yourself when possible
- Include: price comparison, availability, pros/cons

### Service-Based Business

**Most effective**: Guides, How-tos, Comparisons
- Target "how to [accomplish goal]"
- Deep guides establishing expertise
- Comparison articles vs competitors
- CTAs to: consultation, free trial, contact form

### SaaS/Software

**Most effective**: Reviews, Comparisons, How-tos
- Review articles targeting "[Tool] Review"
- Comparison articles targeting "[Tool] vs [Competitor]"
- How-to guides for common use cases
- CTAs to: free trial, pricing page, demo

### Local Services

**Most effective**: Guides, How-tos, FAQ-heavy
- "How to choose [service]" guides
- Local comparison guides
- FAQ articles addressing common concerns
- CTAs to: request quote, schedule consultation

## Writing Tips for Each Type

**For Guides**:
- Comprehensive > short (2,500+ words)
- Make it reference-quality (people link to it)
- Answer every possible question
- Include original research/data when possible

**For Reviews**:
- Personal experience is key (not just specs)
- Be honest about cons (credibility)
- Specific numbers/metrics (not vague praise)
- Clear winner (don't be wishy-washy)

**For Comparisons**:
- Focus on actual differences (not generic features)
- Address the real decision point
- Use tables for scannable comparison
- Don't fake neutrality (recommend one if it's better)

**For How-Tos**:
- Assume reader is beginner (explain everything)
- Step numbers are critical (shows progress)
- Include images/screenshots for each step
- Troubleshooting section is important

**For Roundups**:
- Test/research all products (don't fake it)
- Be consistent in review depth (each product covered equally)
- Real alternatives (not just brand promotions)
- Final verdict clear (best overall, best budget, etc.)

## Common Framework Mistakes

**Mistake 1: Mixing frameworks**
- Half guide, half review
- Results: Confusing structure, doesn't rank for anything

**Correct**: Pick one framework, execute completely

**Mistake 2: No personal experience**
- Rewrite competitor articles
- Results: Thin, duplicate, outranked

**Correct**: Include testing, personal examples, unique data

**Mistake 3: Weak conclusions**
- "Both are good options" or "You choose"
- Results: Low engagement, unclear CTA

**Correct**: Make clear recommendation based on use case

**Mistake 4: Table of contents without jump links**
- Lists sections but links don't work
- Results: Poor UX, lower ranking signals

**Correct**: Every heading should have working jump link

**Mistake 5: Not following framework structure**
- Skip sections ("skipping FAQ because I covered it")
- Results: Incomplete coverage, lower ranking potential

**Correct**: Follow framework completely for best results
