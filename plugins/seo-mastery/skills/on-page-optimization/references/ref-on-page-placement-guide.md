# On-Page Keyword Placement Guide

## The Complete 20 Placement Spots (Hockman)

### Group 1: Meta Tags (3 spots)
1. **Meta title** -- #1 most important on-page factor. Front-load keyword, under 60 chars
2. **Meta description** -- not a ranking factor but impacts CTR. Front-load keyword, under 160 chars
3. **Meta keywords** -- OBSOLETE. Skip entirely

### Group 2: URL (4 spots)
4. **Domain name** -- EMD advantage is real. Keyword in domain = fewer links needed (Morrison evidence: 15 vs 300+ links). The specificity-capital tradeoff: more specific domain = fewer links needed but less expansion room
5. **Subdomain** -- partial match keyword if using subdomains (rare for most sites)
6. **Folder name** -- partial match keywords for categories. Hyphens only. Remove connector words. Never repeat parent folder words
7. **Page name URL** -- exact-match primary keyword. Add secondary keywords. Secret tactic: `/top-best-air-conditioners-reviews` targets multiple keyword combinations simultaneously

### Group 3: Header Tags (2 groups)
8. **H1 tag** -- one per page. Front-load keyword. 20-70 chars. Near-identical to meta title
9. **H2-H6 tags** -- H2/H3 most powerful. Main keyword in first AND last H2. Secondary keywords in subsequent H2s. H3s for variations/synonyms. 20-60 chars

### Group 4: Body Content (3 areas)
10. **Introduction (first 100 words)** -- ALL keywords present. Main keyword in first 2-3 sentences
11. **Main content** -- each keyword 2-3x minimum (under 1K words), 4-5x for longer. Spread evenly across beginning, middle, end
12. **Summary (last 100 words)** -- ALL keywords present. Position within summary is flexible

### Group 5: Images (4 spots)
13. **Alt text** -- exact-match keyword. Descriptive. Front-loaded. Max 125 chars. No "image of" prefix
14. **Image title attribute** -- copy from alt text
15. **Image file name** -- exact-match keyword with hyphens. JPEG format for metadata editing
16. **Image metadata** -- keywords field + description field. Use Photoshop, GIMP, or thexifer.net

### Group 6: Other HTML (4 spots)
17. **Title attributes on HTML elements** -- apply to already-optimized elements. Use partial match keywords on others. Supplemental optimization only
18. **CSS class names** -- single-source claim (Hockman/PBSKids evidence). Global classes: 1-2 word root keyword. Page-level classes: target keyword. Check competitors first. Start small (3-5 instances) if testing
19. **Internal link anchor text** -- exact-match and partial-match keywords. Max 5 words per anchor. Must link from topically relevant pages
20. **Navigation menu link text** -- signals most important pages to Google. Exact-match for unique keywords per page cluster. Partial-match when cluster shares a core keyword

## Header Tag Rules (Complete)

### H1 Tag
1. Include main keyword as exact match
2. Front-load the keyword
3. Add secondary keywords if natural
4. 20-70 characters
5. Meet user intent -- H1 must match what searcher expects
6. Only ONE H1 per page
7. Near-identical to meta title (prevents Google rewrites)

### H2 Optimization
1. Main keyword in FIRST H2 (exact match, front-loaded)
2. Main keyword in LAST H2 (end every page with an H2 summary)
3. Keyword position matters -- closer to start = more weight
4. Secondary keywords in subsequent H2s (one per H2, no repeats)
5. Partial match keywords in additional H2s to avoid over-optimization
6. 20-60 characters
7. You can never have too many H2s and H3s

### H3 Optimization
- Keyword variations and synonyms for main/secondary keywords
- Example: main keyword "rhinoplasty surgery cost" -> H3s use "nose job fees", "nose correction"

### E-Commerce Header Structure (Cromwell)
- H1 = target keyword (only one)
- H2 = below-the-fold heading, different wording from H1
- H3s = three biggest unique selling points of the product category

### Question-Style H2s for AI (brightonSEO/Novitovic)
- Effective for AI Overview extraction
- Keep answer blocks to 40-60 words under each question H2
- Do not force every H2 into question format -- mix naturally

## URL Optimization (Complete)

### Domain Name Strategy (Morrison)

| Strategy | Example | Best For | Link Scaling |
|----------|---------|----------|-------------|
| Exact match keyword | carinsurancereviews.com | One specific phrase | ~10 links |
| Partial match | insurancereviews.com | Broader category | ~20-30 links |
| Single topic | insurancepros.com | Niche dominance | ~40-50 links |
| Generic branded | theinvestingguy.com | Maximum flexibility | ~100-300 links |
| Random brand | gregsrandomsite.com | Pure brand play | ~1,000+ links |

### Folder Name Rules
1. Partial match keywords reflecting top-level categories
2. Always hyphens, never underscores (Google treats hyphens as word separators)
3. Remove numbers and connector words ("for", "the", "and", "a")
4. Never repeat words from parent folders

### Page Slug Rules
1. Exact-match primary keyword
2. Add secondary keywords for multi-keyword targeting
3. Hyphens between words
4. Remove dates/numbers unless keyword includes them
5. 5-7 word semantic slugs get 11.4% more AI citations (Profound)

## Navigation Menu Optimization (Hockman)

### Why Navigation Links Matter
- Signal to Google which pages are most important
- More times a page is linked = more authority received
- NerdWallet ranks #1 for multiple high-value terms that are prominently in their nav

### Rules
1. Include your most important (money) pages
2. For pages with NO shared core keyword: use exact-match keyword phrases
3. For page clusters WITH a shared core keyword: use partial matches to avoid over-optimization (drop the shared word from some menu items)

## The 8-Step Article Optimization Checklist (Nat Eliason)

1. **Keyword-focused slug** -- only primary keyword, no dates, no categories
2. **Keyword + solution focused title** -- keyword + compelling reason to click
3. **Keyword + solution focused meta description** -- make people want to click
4. **Link to your other articles** -- internal linking pass before publishing
5. **Good external links** -- authoritative sources (Wikipedia, academic, .gov, NCBI, major publications). Never other blogs as sources
6. **Right headers** -- one H1, H2s for sections, H3s for subsections, keywords natural in some headers
7. **Image alt tags** -- every image must have descriptive alt text
8. **Link from old articles to new** -- after publishing, add links from existing articles back to the new one

## E-Commerce On-Page Specifics

### Collection Description Split (Cromwell)
Problem: Shopify puts descriptions above product grid, pushing products below fold.
Solution: ~150 words above grid (keyword in first sentence) + 350+ words below via theme code split. Google reads all content the same way.

### Keyword Density for E-Commerce (Cromwell)
- ~500 word collection description: 4-6 mentions of main keyword + semantic/secondary throughout
- Do NOT bold, italicize, or capitalize target keywords (contributes to spam signals)
- Focus on natural placement over density metrics

### Sub-Collection Revenue Multiplier (Cromwell)
Sell same products across multiple collections, each targeting different keywords.
Example: "Natural Sunscreen" (9,900/mo) + "Natural Sunscreen Lotion" + "Natural Sunblock" (900/mo) -- same 5 products, 3-4x the keyword coverage.
