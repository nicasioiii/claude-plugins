# PBN Construction & Management

## Domain Acquisition Process

### Morrison's Filter Setup (Register Compass)
- TLD: .com, .net, .org only. Skip .info (spammy)
- Auction type: Public and Pre-Release only. Skip Buy Now/Offer (overpriced)
- Trust Flow: minimum 7-8
- Domain Authority: minimum 11-13
- Pro tip: filter for domains with at least 1 .gov or .edu link

### Domain Name Quick-Check
- Domain name should logically match category (motorcycle domain in Recreation/Motorcycles = good)
- DA and PA within ~12-14 of each other (20+ gap = likely not quality)

### Backlink Analysis Checklist
1. Check titles/URLs of referring domains for common theme (no theme = skip immediately)
2. Verify DR AND domain traffic alignment (DR 60-70 should show 1,000+ traffic)
3. Look for geographic relevance bonuses
4. Moz Spam Score: max 30% (some good domains reach 40% in certain niches)

### Diggity Domain Studies (Referenced by Floate)
- Transfer backorder domains to premium registrar upon purchase
- Let expired domains sit for 35 days before linking
- Keeping content consistent via Wayback restoration increases domain pass rate
- Traffic is "nice-to-have" not "must-have" -- metrics matter more

## PBN Setup

### Hosting Rules
- NEVER use SEO hosting or "PBN hosting" (except LaunchCDN)
- Each domain: different provider, account, identity, email
- Recommended hosts: SiteGround, HawkHost, Hostinger, GreenGeeks, NameCheap Hosting, HostArmada, iPage
- Each domain needs unique IP address

### CDN Diversification
- Cloudflare (free): own account per domain, delete unnecessary DNS records
- Amazon CloudFront as alternative
- LaunchCDN: truly randomized CDN portfolio ($49/month for 20 sites)
- 50-site split: 10 Cloudflare, 10 CloudFront, 30 premium shared
- CrimeFlare can expose Cloudflare-hidden domains -- reason to diversify CDNs

### Complete Anti-Footprint Checklist

**Domain level:**
- Multiple registrars (GoDaddy, Namecheap, Dynadot)
- Different contact info per domain (Fake Name Generator)
- Different email addresses per registration
- Register at different times/dates
- Different nameservers (via different hosts + CDNs)
- Mix TLDs: .com, .org, .net, .info

**Hosting level:**
- Never PBN/SEO hosting providers
- Different hosting companies
- Unique IP per domain
- Servers in different locations

**Backend level:**
- Fix 404 pages using Wayback Machine (don't redirect all to homepage)
- DO let crawlers (Ahrefs, Moz, SEMrush) scan your sites -- blocking is MORE suspicious
- Different Google Analytics accounts (or alternatives: Clicky, Open Web Analytics)
- Different plugins per site (Yoast, RankMath, AIOSEO, Squirrly)
- Different caching plugins (WP Fastest Cache, W3 Total Cache, Autoptimize)
- Different WordPress themes from different theme shops
- Different author names per site
- Legal pages with different naming conventions

**Content level:**
- At least 5 planned categories per site
- Max 1 outbound link per 250 words
- Homepage shows excerpts or full posts (not full text of all articles)
- Social profiles for each PBN
- Identifiable author (not "admin")
- Different word counts, posting times, posting days
- NEVER interlink PBNs to each other
- Vary outbound link patterns

## PBN Content Standards

### Minimum Requirements
- 3-4 articles per PBN minimum (Morrison), ideally 5-20
- Articles ~1,000 words each
- Content needs to be passable and readable, not top-notch

### Content Sources (Cheapest to Most Expensive)
1. Wayback Machine restoration (recover cached pages with backlinks)
2. AI content (Article Forge or any AI writer -- fine for PBNs)
3. Content spinning (WordAI, SEO Content Machine, Spin Rewriter)
4. Content curation (handpick + commentary via Feedly)
5. Hired writers (iWriter for cheap, Upwork for quality)

### Appearance Note (Morrison)
PBN site appearance doesn't matter. All penalties traced were algorithmic (same C-class IP, too many OBLs, linking same sites together) -- never from site appearance. Blog roll on homepage works fine.

## PBN Linking Rules

- 1 link per 750-1,000 words of content (includes all outbound links)
- 1-3 outbound authority links per money site link as camouflage
- Find authority OBLs: Google the target term, pick a top-10 result
- Can link to multiple money sites from one PBN (mixing confuses Google)
- Social media platform links don't count as "money site" links
- Too many OBLs per content volume = penalty trigger

### Homepage Link Power
- Always ensure money link appears on PBN homepage
- Choose themes that display full posts on homepage (not excerpts)
- If theme shows snippets, your link is on an inner page = far less power

### Anchor Text for PBNs (Morrison)
- Always use keyword anchor text on PBN links
- Use money keyword or close long-tail variations
- Save branded anchors for other link types (citations, socials, guest posts)

## PBN Growth & Maintenance

### Reverse Silo Method
- Recover Wayback-cached pages with backlinks as pillar posts
- Create supporting articles targeting related questions
- Internal linking: supporting articles link to pillar post and each other
- Re-optimize with Surfer SEO once pillar posts start ranking

### Simple Link Building for PBNs
- Blog commenting: ~50/week, expect ~20% approval rate (delegate to VA)
- Directory submissions: ~50/month (avoid link exchange or payment directories)
- Social profiles: 10+ social accounts per PBN (NameChk for availability)
- Content syndication: WordPress.com, Tumblr, Blogger, Wix, Weebly, Medium
- Infographics: Canva + embed code, submit to directories

### Diversifying Across PBNs
Each PBN should have completely different approaches to publishing schedule, silo structure, OBL ratio, commenting activity, directory submissions, social profiles, and syndication patterns. No patterns connecting PBNs.

### PBN Recycling (Morrison)
- Strip all old outbound links
- Leave old content up
- Change homepage title to new topic
- Set WordPress to show only 3 most recent (new topic) posts on homepage
- Remove old sidebar content
- Always re-run penalty check before linking again

### Saving Inner Page Power (Morrison)
- Check Ahrefs "Best by Links" for inner pages with referring domains
- Redirect some inner pages to homepage (Redirection plugin) -- not all
- Rebuild other inner pages with AI content about original page's topic
- Mix of redirect and rebuild is safest

## Single-Variable Testing

### How to Test PBN Effectiveness
1. Create a search query with zero results (quoted made-up phrase)
2. Create identical pages on a test site with random URL strings
3. Optimize each page's meta title for the unique keyword
4. Build PBN links with exact-match anchor text
5. Wait at least 30 days

### Interpreting Results
- All pages indexed after 30 days = PBNs are clean
- Ranking position shows relative PBN strength
- History shows which links pass juice fastest
- If test domain gets deindexed = identify and remove the bad PBN

### Morrison's Penalty Power Check
- Google the PBN domain name -- if #1 = passing
- Search "domain name + [terms]" to check how many results Google returns
- More results = stronger domain
- Always recheck before linking from recycled PBNs

## Foreign SEO PBN Building

- Google's English algorithm has had massive investment; foreign languages lag
- German and Spanish approaching English-level scrutiny
- English domains CAN be repurposed for foreign PBN niches (but NOT the reverse)
- Rules: avoid heavy niche anchor text in English profile, no bad neighborhoods, no existing foreign languages in link profile
- May 2020 update targeted foreign SEO -- fix by adding country-specific TLD profile/forum links
